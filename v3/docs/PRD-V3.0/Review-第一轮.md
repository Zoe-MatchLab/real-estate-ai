## 任务状态流转验证

> 本章节为Review重点，对照V3.0解决方案第四章"任务状态流转设计"与V3.0 PRD中的任务状态定义，进行逐项交叉验证。

### 4.1 状态字典对比

**V3.0 Solution定义**（dict_task_status）：

| status_key | 统一展示文本（PC） | 统一展示文本（小程序） | 说明 |
|-----------|-----------------|---------------------|------|
| `pending` | 待执行 | 待执行 | 任务已下发，待开始 |
| `in_progress` | 进行中 | 进行中 | 正在执行，有阶段数据 |
| `paused` | 已暂停 | 已暂停 | 任务暂停（可恢复） |
| `submitted` | 已提交 | 已提交待审核 | 成果已提交，等待审核 |
| `approved` | 已通过 | 已通过 | 审核通过 |
| `rejected` | 已驳回 | 已驳回 | 审核驳回，需整改 |
| `expired` | 已超时 | 已超时 | 超过截止时间未完成 |

**V3.0 PRD定义**（5.2.3探盘任务执行流程）：

| 状态 | 说明 | 流转 |
|------|------|------|

| 已提交 | 成果已提交，等待审核 | → 已通过/已驳回 |
| 已通过 | 审核通过 | — |
| 已驳回 | 审核驳回，需修改 | → 待执行（整改后重新提交） |
| 已逾期 | 超过截止时间 | — |

### 4.3 核心问题

#### 🔴 问题1：状态数量不一致（7 vs 6）

V3.0 Solution定义了7个状态（含`paused`暂停状态），V3.0 PRD仅定义了6个状态（无暂停状态）。

**影响**：
- 若小程序端支持"暂停任务"功能（Solution V3.0-TA-01的WebSocket同步机制中提及），则PRD必须补充暂停状态
- 若PRD不打算支持暂停功能，则Solution的paused状态需删除

**建议**：与业务方确认是否需要在任务执行过程中支持主动暂停功能。

#### 🔴 问题2：待分发状态处理不一致

V3.0 PRD定义了"待分发"状态（下发人配置任务但未推送），Solution无此状态。

**分析**：
- Solution中任务从"待执行"开始，隐含了任务已推送的前提
- PRD的"待分发"表示任务配置完成但尚未推送给执行人

**建议**：明确任务的生命周期起点——是"配置完成即推送"（Solution模式）还是"配置完成需手动推送"（PRD模式）？这影响任务通知的触发时机。

#### 🟡 问题3：状态key命名规范

Solution使用英文status_key（pending/in_progress/submitted等），PRD使用中文状态名称。

**建议**：V3.0 PRD附录补充状态字典表，统一使用status_key作为系统内部标识，中文仅作为展示文本。

## Review结论

### 通过情况

| 类别 | 数量 | 说明 |
|------|------|------|
| ✅ 正常通过 | 2 | 新盘81字段录入（新盘基础闭环）；跟进保存空内容校验（V3-PT-02，纯实现层） |

| ❌ 存在重大矛盾或缺失 | 5（P1） | 录音分析"暂未实现"状态矛盾、任务状态不一致、WebSocket机制缺失、PC端客户管理缺失 |
| 🆕 PRD独有内容 | 1 | 小程序内容日历（5.8章节，Solution未提及但PRD有定义） |

### 总体评估

1. **V3.0 PRD与V3.0 Solution存在结构性Gap**：V3.0 PRD基本沿用V2.0内容，V3.0解决方案中大量增强功能（如PC端训练数据中心、录音督查功能、GPS兜底方案、指标口径变更记录等）未在V3.0 PRD中同步更新。

2. **任务状态流转是最大风险项**：Solution定义7个状态+WebSocket同步机制，PRD定义6个状态+"用户手动刷新"，两者在状态数量、待分发定义、同步技术上均存在不一致，如不统一将导致双端状态显示混乱。

3. **录音分析模块存在根本性矛盾**：V3.0 PRD写"暂未实现"，V3.0 Solution要求实现PC端管理+督查功能，需业务方明确方向。

4. **PC端客户管理模块需要完整补充**：这是V3.0的重大功能增补（#CM-01），涉及列表页+详情页+跟进记录Tab+通话历史Tab+批量转移+权限矩阵，体量较大。

### 优先级处理建议

#### 🔴 高优先级（必须处理，影响V3.0开发启动）

1. **立即确认**：录音分析V3.0是否实现PC端管理功能（P1-01）
   - 若实现：PRD需大幅补充PC端录音分析章节（预计新增3-5页）
   - 若不实现：Solution中的V3-CL-04、V3-CL-05应移出V3.0范围

2. **统一任务状态定义**（P1-02、P1-03）
   - 明确是否需要暂停状态（paused）
   - 明确任务生命周期起点（待分发 vs 待执行）
   - 输出统一的双端状态字典表

3. **补充WebSocket实时同步技术方案**（P1-04）
   - 在V3.0 PRD中增加"技术架构-实时同步"章节
   - 或明确引用V3.0 Solution的技术方案作为开发依据

#### 🟡 中优先级（V3.0开发过程中同步补充）

4. **完整补充PC端客户管理模块**（P1-05）：这是V3.0最大的功能增补，建议单独作为一个Sprint或功能包处理

5. **按模块同步Solution内容到PRD**：P2-01～P2-12共12项功能闭环缺陷，建议按模块分批补充到V3.0 PRD中
