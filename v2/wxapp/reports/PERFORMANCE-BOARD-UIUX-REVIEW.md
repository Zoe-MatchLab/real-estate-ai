├─────────────────────────────────────────┤
│  ┌─────────┐  ┌─────────┐           │
│  │ 📊发布  │  │ 💬咨询  │           │ ← KPI 概览
│  │   28    │  │   156   │           │
│  └─────────┘  └─────────┘           │
│  ┌─────────┐  ┌─────────┐           │

│  │   23    │  │   18    │           │
│  └─────────┘  └─────────┘           │
├─────────────────────────────────────────┤
│  内容发布                               │
│  发布量 │ 浏览量 │ 投流量             │

├─────────────────────────────────────────┤

│  咨询开口成本 │ 带看成本 │ 目标对比     │

**产品评审意见：**
> 1. KPI 概览是否需要显示目标对比（如：发布量 28/目标20）？
> 2. 是否需要显示「成交」指标？
> 3. 各模块的「详情」入口是否需要？

**UX 评审意见：**
> 1. 漏斗图的宽度是否一致？
> 2. KPI 卡片是否需要更明显的颜色区分？
> 3. 模块之间的间距是否需要调整？

### 3.3 交互设计

| 检查项 | 当前实现 | 评审意见 |
|--------|----------|----------|
| 时间筛选 | ✅ Tab 切换 | 可增加确认操作 |
| 按压反馈 | ✅ scale(0.98) | 一致 |
| 跳转逻辑 | ✅ Toast 提示 | 所有卡片可点击 |
| 空状态 | N/A | 暂不需要 |

**UX 评审意见：**
> 1. 点击 KPI 卡片和点击「详情」的区别是什么？
> 2. 是否需要下拉刷新数据？

### 3.4 可访问性

| 检查项 | 当前实现 | 评审意见 |
|--------|----------|----------|
| Focus 样式 | ✅ | 已添加 |
| 按钮 label | ⚠️ | 部分按钮缺少 aria-label |
| 颜色对比 | ✅ | WCAG 2.1 AA 达标 |

**前端评审意见：**
> 建议为所有可点击元素添加 aria-label：
> ```html
> <div class="kpi-card" onclick="..." aria-label="查看内容发布详情">
> ```

### 3.5 数据可视化

| 检查项 | 当前实现 | 评审意见 |
|--------|----------|----------|
| 漏斗图 | ✅ 4层完整 | 宽度比例清晰 |
| 数据网格 | ✅ 3列布局 | 适合移动端 |
| 颜色编码 | ✅ 按类型区分 | 橙/绿/紫/蓝 |

**UX 评审意见：**
> 1. 漏斗图是否可以点击查看明细？
> 2. 是否需要趋势箭头（如 ↑12%）？
> 3. 数字是否需要千分位格式化（如 1,200）？

## 八、附录

### 8.1 SOP 指标定义

| 指标 | 定义 | 计算公式 |
|------|------|----------|
| 发布量 | 各平台每日内容发布数量 | 合计 |
| 浏览量 | 各平台内容曝光量 | 合计 |
| 投流量 | 各平台付费推广流量 | 合计 |
| 咨询量 | 客户主动咨询总次数 | 合计 |
| 线索量 | 留资客户数量 | 合计 |
| 报备量 | 客户报备楼盘数量 | 合计 |
| 带看量 | 经纪人带客户看房次数 | 合计 |
| 触达量 | 主动触达老客次数 | 合计 |
| 咨询开口成本 | 单个咨询成本 | 投流量/咨询量 |
| 带看成本 | 单次带看成本 | 投流量/带看量 |

### 8.2 专家等级定义

| 等级 | 定义 | 衡量标准 |
|------|------|----------|
| 单盘专家 | 精通单个楼盘 | 该楼盘话术训练≥90分 |
| 商圈专家 | 精通商圈内80%楼盘 | 商圈内80%楼盘达标（≥80分） |
| 行政区专家 | 精通行政区所有楼盘 | 行政区内80%楼盘达标 |
| 城池专家 | 精通全城楼盘 | 全城80%楼盘达标 |
