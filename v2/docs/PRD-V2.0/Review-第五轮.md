## 二、核心业务流程完整性验证

### 2.1 全流程链路验证

**业务流程总览：**

```
[新盘录入] → [提交审核] → [审核通过] → [新盘上线]
                                              │
                            ┌─────────────────┼─────────────────┐
                            ↓                 ↓                 ↓
                      [通知可下发人员]   [自动关联训练题库]   [沉淀数据资产]
                            │
                            ↓
                      [下发探盘任务]
                            │
                            ↓
                      [通知执行人员]
                            │
                            ↓
    ┌───────────────────────┼───────────────────────┐
    ↓                       ↓                       ↓
[探盘任务]            [话术训练]              [约看录音分析]
(4阶段并行)           (3阶段并行)              (独立并行)
    │                       │                       │
    ↓                       ↓                       ↓
[物料库汇聚] ←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←
    │
    ↓
[内容发布] → [获客转化] → [数据看板]
```

**验证结论：✅ 全链路无断点**

- 新盘上线触发三个并行任务（探盘/训练/录音），互不阻塞
- 物料库作为所有任务的汇点，统一沉淀探盘成果和AI生成内容
- 数据看板依赖上游全量数据，形成完整的业务闭环
