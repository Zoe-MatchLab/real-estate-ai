## 一、项目概述

### 1.1 我们做什么

房产AI作业平台是一款服务于新房业务的**SOP智能操作系统**，核心理念四个字：**探-练-推-约**。

### 1.2 解决什么问题

传统模式下，经纪人做新房业务面临四大困境：

| 困境 | 现状 | 平台如何解决 |
|------|------|------------|
| **信息获取滞后** | 经纪人依赖线下踩盘或转发资料，无法快速全面了解楼盘 | 标准化新盘数据库，81个字段结构化沉淀，实时同步更新 |
| **能力参差不齐** | 产品知识掌握程度不一，约看率、成交率不稳定 | AI话术训练，实时评分反馈，能力可量化追踪 |
| **物料质量不一** | 探盘成果标准不统一，生成的新媒体内容质量难以保障 | AI自动生成高质量文案/图片/视频，审核后统一物料库 |
| **管理抓手缺失** | 管理层难以实时掌握经纪人任务执行进度，缺乏数据支撑 | 实时数据看板，任务全链路追踪，团队业绩透明化 |

### 1.3 核心价值主张

> **「探得真材，炼出专业，推向市场，约见成交」**

| 价值维度 | 具体描述 |
|----------|---------|
| **新媒体获客** | 通过探盘获取真实物料，AI生成小红书、公众号等平台高质量内容，实现线上获客 |
| **产品力提升** | 基于楼盘的话术训练，帮助经纪人快速掌握产品卖点，提高约看率和成交率 |
| **管理可视化** | 实时追踪任务执行进度，数据看板辅助管理层科学决策，沉淀数据资产 |

### 2.2 功能亮点

#### 新盘管理
- **81字段结构化录入**：基本信息、项目规格、开发商物业、楼盘特色、交通/商业/教育/公园/医疗配套、会所配套、价格动态、竞品信息、项目定位全覆盖
- **动态字段同步**：变更日志+版本号机制，确保小程序端实时感知新盘信息更新
- **自动关联**：新盘上线自动关联训练题库、自动通知可下发人员

#### 探盘任务
- **4阶段标准化流程**：定位打卡 → 点位采集 → 文案确认 → 封面/视频生成
- **PC端结构化驳回**：5类18项驳回原因，驳回信息结构化传递到小程序端
- **审核辅助工具**：PC端可见打卡轨迹、过程数据，审核更客观

#### 话术训练
- **AI对话式练习**：模拟真实客户问答，实时反馈改进方向
- **数据双端回流**：训练记录同步至PC端，管理层可查看团队训练情况
- **题库智能关联**：标签多对多匹配+专属题库+优先级机制

#### 录音分析
- **PC端查看**：管理者可查看经纪人录音分析结果，无需登录小程序
- **三维分析**：话术技巧 + 情绪判断 + 客户需求 + 改进建议
- **客户关联**：录音可关联至客户，支持在客户详情页查看通话历史

#### 物料管理
- **统一物料库**：探盘原图、AI生成文案/图片/视频素材统一管理
- **小程序物料中心**：TabBar入口，经纪人可随时查看/下载/分享
- **小程序码海报**：物料可生成带参数小程序码，支持按来源区分跳转**

#### 数据看板
- **指标双端统一**：PC与小程序使用同一套metrics_definition配置表，口径完全一致
- **三层刷新机制**：实时（≤5分钟）/日度（T+1）/历史预计算
- **能力雷达图**：单盘专家 → 商圈专家 → 行政区专家 → 全城专家

#### 客户管理（V2.0新增）
- **PC端完整功能**：客户列表/详情/编辑/转移，跟进记录，关联录音
- **批量转移**：支持客户批量转移，主管可统一分配客户资源

### 3.2 三大任务并行机制

| 任务类型 | 是否必选 | 与其他任务关系 | 核心价值 |
|----------|---------|--------------|---------|
| **探盘任务** | ✅ 必选 | 核心任务，其他任务可并行 | 获取真实物料，统一内容标准 |
| **话术训练** | 可选 | 独立并行，不依赖探盘完成 | 快速掌握产品卖点，提升邀约能力 |
| **约看录音分析** | 可选 | 独立并行，不依赖探盘完成 | 复盘话术效果，持续优化改进 |

> **设计理念**：三大任务独立并行，互不阻塞，最大化经纪人作业效率。

### 3.3 探盘任务4阶段详解

```
阶段1：定位打卡
│
├── 到达楼盘现场
├── 系统自动定位校验（需在打卡范围内）
└── 签到成功，记录打卡时间和位置
        │
        ↓
阶段2：点位采集
│
├── 按配置的打卡点位逐一完成资料采集
├── 支持：拍照📷 / 视频🎬 / 录音🎤 / 文本📝
├── 必填点位不可跳过，可选点位可跳过
└── 所有必填点位完成 → 自动解锁下一阶段
        │
        ↓
阶段3：文案确认
│
├── AI自动生成：小红书文案/抖音脚本/朋友圈文案/短视频口播
├── 经纪人查看/编辑/确认
└── ⭐ 确认后任务即算完成（图片/视频后台持续生成）
        │
        ↓
阶段4：封面/视频生成（后台执行）
│
├── AI生成封面图 + 短视频
├── 生成过程不阻塞用户操作
├── 生成完成后推送通知
└── 确认成果 → 提交审核
```

### 5.3 数据资产沉淀

| 资产类型 | 具体形态 | 价值 |
|---------|---------|------|
| **楼盘基础数据** | 81字段+图片库 | 统一楼盘信息标准，消除信息差 |
| **探盘物料** | 原图、视频、AI生成文案/图片/视频 | 沉淀真实物料资产，赋能全员 |
| **训练数据** | 训练记录、评分记录、能力维度 | 量化经纪人能力，追踪提升轨迹 |
| **获客数据** | 发布记录、咨询/线索/报备/带看/成交 | 评估获客ROI，优化投放策略 |

### 5.4 数据双端一致性保障

| 保障机制 | 说明 |
|---------|------|
| **统一数据源** | PC端和小程序端共用同一数据库，无独立数据源 |
| **统一指标服务** | PC端和小程序端共用同一套metrics_definition配置表 |
| **实时同步** | 探盘打卡数据、训练记录实时同步至PC端 |
| **版本号机制** | 新盘字段变更带版本号，小程序端自动检测更新 |
| **变更日志** | 所有数据变更记录在building_change_log表 |
