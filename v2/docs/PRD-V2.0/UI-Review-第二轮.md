# UI/UX Review 第二轮报告

> 评审日期：2026-04-12
> 评审人：UI/UX 设计师
> 版本：V2.0 HTML 原型
> 评审重点：**跨端一致性 + 无障碍设计**

---

## 一、跨端一致性检查

### 1.1 品牌色对比 ⚠️ 严重不一致

| 端 | 文件 | primary 色值 | 颜色 |
|---|------|-------------|------|
| PC端 | 全部页面 | `#165DFF` | 蓝色 |
| 小程序 | `manager-index.html` | `#165DFF` | 蓝色 |
| 小程序 | `task-list.html` | `#fa8c16` | 橙色 |
| 小程序 | `speech-train.html` | `#FA8C16` | 橙色 |
| 小程序 | `train.html` | `#fa8c16` | 橙色 |

**问题**：
- 小程序端内部品牌色就不统一！
- `manager-index.html`（管理者工作台）用蓝色
- `task-list.html`、`speech-train.html`、`train.html` 用橙色
- 建议：**同一产品应使用统一的品牌色**，要么全是蓝色系，要么全是橙色系

### 1.2 语义色统一性 ⚠️ 基本一致

| 语义 | PC端 | 小程序端 |
|------|------|---------|
| 成功 | `#00B42A` | `#00B42A` ✅ |
| 警告 | `#FF7D00` | `#FF7D00` ✅ |
| 危险 | `#F53F3F` / `#FA5151` | `#FA5151` ✅ |
| 信息 | `#165DFF` | 各自不同 ❌ |

### 1.3 状态标签类名不一致 ⚠️ 需统一

| 状态 | PC端类名 | 小程序端类名 |
|------|---------|-------------|
| 高优先级/危险 | `.tag-level-high` | `.tag-danger` |
| 中等/警告 | `.tag-level-medium` | `.tag-warning` |
| 低优先级/成功 | `.tag-level-low` | `.tag-success` |
| 信息/默认 | `.tag-level-deal` | `.tag-info` |

**问题**：两端无法共用同一套 CSS，组件不可复用

### 1.4 术语命名不一致 ⚠️ 需对齐

| 功能 | PC端 | 小程序端 |
|------|------|---------|
| 任务列表 | 探盘任务列表 | 我的任务 |
| 训练模块 | 训练数据中心 | 话术训练助手 / AI训练中心 |
| 客户详情 | 客户详情 | （未发现对应页面） |

**建议**：建立统一的术语词典，明确"探盘任务"是否等同于"我的任务"

## 二、无障碍设计（Accessibility）检查

### 2.1 屏幕阅读器支持 ❌ 全面缺失

| 检查项 | PC端 | 小程序端 | 说明 |
|--------|------|---------|------|
| 图标按钮 aria-label | ❌ 未使用 | ❌ 未使用 | 所有 `i.fa-xxx` 均无 aria-label |
| 表单 label 关联 | ❌ 未使用 | ❌ 未使用 | 全部使用 placeholder 代替 |
| 角色属性 role | ❌ 未使用 | ❌ 未使用 | 按钮、导航等无语义化 role |
| 焦点可见性 | ⚠️ 部分 | ⚠️ 部分 | focus:ring 样式存在但不统一 |
| 跳过导航链接 | ❌ 无 | ❌ 无 | 无"跳到内容"链接 |

**典型问题示例**：
```html
<!-- 图标按钮无 aria-label -->
<button class="text-gray-600 hover:text-primary relative">
    <i class="fa fa-bell-o text-lg"></i>  <!-- 屏幕阅读器无法识别"通知" -->
</button>

<!-- 表单使用 placeholder 代替 label -->
<input type="text" placeholder="搜索客户姓名或手机号...">
<!-- 屏幕阅读器不知道这是"客户搜索"字段 -->
```

### 2.2 颜色对比度 ⚠️ 部分不达标

| 颜色值 | 背景色 | 对比度 | WCAG AA 要求 | 状态 |
|--------|--------|--------|-------------|------|
| `#86909C` | `#FFFFFF` | 3.5:1 | 4.5:1 | ❌ 不达标 |
| `#4E5969` | `#FFFFFF` | 5.2:1 | 4.5:1 | ✅ 达标 |
| `#00B42A` | `#E8FFEA` | 4.8:1 | 4.5:1 | ✅ 达标 |
| `#FF7D00` | `#FFF7E8` | 3.8:1 | 4.5:1 | ⚠️ 边缘 |
| `#F53F3F` | `#FFECE8` | 4.2:1 | 4.5:1 | ⚠️ 边缘 |

**问题**：
- 灰色辅助文字 `#86909C` 在白底上对比度不足
- 警告色背景组合（橙色、红色标签）的对比度也偏弱

### 2.3 焦点顺序 ⚠️ 未定义

- 所有页面无明确的 `tabindex` 属性
- 焦点顺序依赖浏览器默认行为，未经验证
- Modal 弹出后焦点未锁定在 Modal 内

## 三、文案规范检查

### 3.1 按钮文案 ✅ 基本规范

| 按钮类型 | 文案 | 评价 |
|---------|------|------|
| 主要操作 | "确认"、"提交" | ✅ 清晰 |
| 次要操作 | "取消"、"重置" | ✅ 清晰 |
| 批量操作 | "批量转移" | ✅ 清晰 |
| 辅助操作 | "导出"、"下载" | ✅ 清晰 |
| 链接型按钮 | "详情"、"跟进" | ✅ 简洁 |

### 3.2 提示文案 ⚠️ 需优化

| 位置 | 当前文案 | 建议优化 |
|------|---------|---------|
| 空状态 | 无 | 建议增加"暂无数据"插画+文案 |
| 搜索框 placeholder | "搜索客户姓名或手机号..." | ✅ 可接受 |
| 表单 placeholder | "请详细描述本次跟进内容..." | ✅ 友好 |
| 弹窗提示 | `alert()` 纯文本 | 建议使用 Toast 轻提示 |

### 3.3 错误文案 ❌ 过于简单

| 当前 | 问题 |
|------|------|
| `alert("请输入转移备注")` | 语气生硬，像系统报错 |
| `alert("操作成功")` | 应使用成功 Toast |
| 无字段级错误提示 | 用户不知道哪个字段出错 |

**建议**：
- 错误提示改为"请填写转移备注，以便接收人了解情况"
- 成功提示改为 Toast 轻提示"客户转移成功"
- 字段错误改为内联提示（如红色小字在输入框下方）

## 五、修复建议

### 5.1 品牌色统一（优先级：P0）

**方案**：统一为蓝色系 `#165DFF`

理由：
- PC端全部使用蓝色
- 蓝色更符合"科技/数据平台"的定位
- 小程序端 `manager-index.html` 已使用蓝色

**操作**：
```js
// 统一修改小程序端的 tailwind.config
colors: {
    primary: '#165DFF',  // 全部改为蓝色
}
```

### 5.2 无障碍改造（优先级：P0）

**图标按钮添加 aria-label**：
```html
<button aria-label="查看通知">
    <i class="fa fa-bell-o"></i>
</button>
```

**表单添加 label 或 aria-label**：
```html
<!-- 方案1：使用 label 元素 -->
<label for="search-customer" class="sr-only">搜索客户</label>
<input id="search-customer" type="text" placeholder="搜索客户姓名或手机号...">

<!-- 方案2：使用 aria-label -->
<input type="text" placeholder="搜索客户姓名或手机号..." aria-label="搜索客户">
```

**颜色对比度修复**：
```css
/* 将 #86909C 改为更深的灰色 */
.text-gray-500 {
    color: #4E5969;  /* 对比度 5.2:1 */
}
```

### 5.3 术语词典建立（优先级：P1）

建议产品侧确认后同步给设计和开发：

| 英文 | 建议中文名 | 适用端 |
|------|-----------|--------|
| Task List | 探盘任务列表 | PC端 |
| My Tasks | 我的任务 | 小程序端 |
| Speech Training | 话术训练 | 通用 |
| Training Data Center | 训练数据中心 | PC端 |

## 六、结论

**整体评价：跨端一致性存在严重问题，小程序端自身品牌色就不统一；无障碍设计全面缺失，不符合 WCAG 基础标准。**

**核心风险：**

1. **品牌色断裂**：小程序端 `manager-index.html` 和其他页面使用不同品牌色，用户在管理者工作台和经纪人工作台之间切换时会感受到明显的视觉跳脱
2. **无障碍空白**：所有页面无任何屏幕阅读器支持，障碍用户完全无法使用
3. **对比度不足**：灰色辅助文字在白底上对比度不达标，影响视力弱用户

**建议处理顺序：**
1. **P0-1**：统一小程序端品牌色（改动最小，影响最大）
2. **P0-3**：修复 `#86909C` 颜色对比度
3. **P0-2**：为图标按钮添加 aria-label（快速可执行）
4. **P1**：建立术语词典，统一状态标签类名
5. **P2**：完善其他无障碍特性
