## 二、配置项分组（页面结构）

### 分组1：楼盘信息（只读展示区）
> 已上线不可修改，作为配置参考

- 楼盘名称、楼盘地址、均价、开发商

---

### 分组2：项目政策
- 政策有效期开始 / 结束（日期选择，可为空表示长期有效）
- 政策描述（文本域）
- 特殊说明（文本域）

---

### 分组3：任务配置

#### 3.1 路书任务

**路书生成方式**：自动生成 / 手动上传（二选一）

- **「自动生成」时**，以下内容由AI自动填充（可编辑确认）：
  - 项目概述（区位、开发商、产品类型等）
  - 价格户型（均价、总价区间、户型面积等）
  - 周边配套（教育/医疗/商业/交通/公园）
  - 开发商介绍
  - 项目亮点
  - **探盘动线**（入口→接待处→沙盘→户型模型→样板间→区位图→洽谈区等节点）
  - **话术节点**（每个动线节点的讲解话术）
  - 竞品对比话术

- **「手动上传」时**，显示文件上传区域，支持PDF/Word/图片格式

**点位物料自定义配置**（当路书生成方式=自动生成时必填）

- 每个采集点位可配置：
  - 物料标题（必填）：如「接待处介绍」「沙盘讲解要点」
  - 物料描述（必填）：详细描述/话术文本
  - 物料附件（选填）：支持文本/语音/图片/视频类型

**采集点位配置**（当路书生成方式=自动生成时必填）

- 采集点位列表（默认6个，支持增删、拖拽排序）
  - 每个点位包含：
    - 点位名称（必填，如「接待处」「沙盘区」「样板间A户型」）
    - 点位类型（必选：必打卡点 / 选打卡点）
    - 采集内容要求（必填：需拍摄照片 / 录制视频 / 填写内容 / 无需采集）
    - 打卡顺序（必填，数字，自动生成，支持拖拽调整）
    - 建议停留时长（选填，分钟）
  - 「重置为默认模板」按钮：一键恢复系统默认的6个标准点位配置

**内容生成维度配置**（当路书生成方式=自动生成时必填）

- 多选框列表，默认全选：
  - [x] 项目亮点
  - [x] 户型分析
  - [x] 周边配套
  - [x] 竞品对比
  - [x] 样板间讲解
  - [x] 购房话术
  - [ ] 物业介绍
  - [ ] 装修标准

**路书内容审核方式**

- 单选：自动通过（默认）/ 人工审核
- 说明文案：自动通过=AI生成后直接下发；人工审核=生成后需管理员确认再下发

**路书有效期**

- 单选：与任务执行时间一致（默认）/ 自定义结束日期
- 当选择自定义时，显示日期选择器

#### 3.2 打卡配置

- 启用探盘任务（总开关）
- 打卡范围（米）：数字输入，默认200
- 偏移阈值说明（三档文案，自动展示）：
  - ≤200m → 正常打卡
  - 200-500m → 警告（记录但标记异常）
  - >500m → 拦截（不允许打卡）
- 可配置拦截阈值：滑动条/数字输入，可在[200m, 1000m]范围内调整，步进100m，默认500m
- 允许补卡：开关，默认关闭
  - 当开启时，显示：
    - 补卡时限（小时）：数字输入，默认24
    - 补卡次数上限：数字输入，默认1
- 打卡照片要求：单选，必填（默认）/ 选填 / 无需拍照
- 打卡话术要求：开关，默认关闭；开启后要求经纪人在打卡时录制语音话术

#### 3.3 任务项配置（通用）

> 统一配置各子任务的启用/必填状态及最少执行数量，来源于PRD 3.4「任务项配置」规则

- **打卡任务**：开关，默认开启
  - 当设为必填时，需配置「打卡最少执行数量」：经纪人至少完成多少个点位打卡才算有效
- **路书任务**：开关，默认开启
- **训练任务**：开关，默认关闭
- **录音任务**：开关，默认关闭
  - 当设为必填时，需配置「录音最少执行数量」：经纪人至少上传并分析多少条录音才算有效

#### 3.5 话术训练配置

- 启用话术训练（开关）
- **选择题库（多选）**：勾选框列表，支持从题库列表中选择多个题库；显示已选数量
- 已选题库展示：显示已选择的题库名称和数量

#### 3.6 邀约录音上传配置

- 启用录音分析（开关）
- 分析维度（多选）：话术规范、情绪识别、关键词覆盖、邀约意向
- AI重试机制：**不需要**（不在页面展示）

## 三、探盘任务规则补充说明

> 以下规则来源于PRD 3.4章节「探盘任务规则」，需在配置说明及前端交互中体现：

### 1. 【探盘路书】
- ⭐ **文案确认后即算完成任务**：图片/视频耗时长，不阻塞任务完成
- **重做任务**：允许（因素材问题效果不佳可重做整个任务）
- **重新生成**：文案确认后不可编辑物料，如需编辑只能重做整个任务

### 2. 产品力训练规则
- 可配置多个题库（选择题库支持多选）
- 首次训练按配置顺序解锁题库

### 4. 约看录音分析规则
- 可选/必填；必填时需配置最少录音数量

### 5. 下发规则
- 新盘上线时配置的可下发人员可下发任务
- 奇点管理员可下发任务

### 6. 可见范围规则
- 可按组织架构或指定人员配置探盘任务成果的可见范围

## 六、问题1结论：楼盘列表审核操作

### 现状分析
当前 `property-list.html`（v3）表格中：
- **待审核**行：已有内联「通过」「驳回」按钮 ✓
- **录入中**行：只有「编辑」「查看」，缺少提交审核入口 ← 缺失点

### 建议方案

#### 列表操作列按钮规则

| 状态 | 操作按钮 |
| --- | --- |
| 录入中 | 编辑、查看、**提交审核** |
| 待审核 | **审核**（点击后展开通过/驳回）、查看 |
| 已驳回 | 编辑、查看驳回原因 |
| 已上线 | 查看、下线、配置 |
| 已下线 | 查看、重新上线 |

> ⚠️ 不需要独立的「重新提交」按钮

#### 编辑页面提交按钮逻辑
- 楼盘状态为「已驳回」时，进入编辑页面后：
  - 底部提交按钮文案显示为 **「重新提交」**
  - 其他状态（录入中首次提交）：按钮文案为 **「提交审核」**

#### 审核流程

```
录入中 → [提交审核] → 待审核 → [审核通过] → 已上线
                              ↘ [审核驳回] → 已驳回
```

#### 审核操作入口

| 入口 | 位置 | 适用场景 |
| --- | --- | --- |
| 列表操作列 | property-list.html 表格「操作」列 | 快速审核（通过/驳回），待审核行显示「审核」按钮 |
| 详情页Tab | material-detail.html 新增「审核」Tab | 查看审核历史、填写审核意见 |
| 详情页顶部 | material-detail.html 操作区 | 快捷通过/驳回按钮 |

#### 审核状态定义

| 状态 | 含义 | 可执行操作 |
| --- | --- | --- |
| 录入中 | 楼盘信息录入中，尚未提交 | 编辑、查看、提交审核 |
| 待审核 | 已提交，等待审核人员审批 | 查看详情、审核通过、审核驳回 |
| 已驳回 | 审核不通过，需修改后重新提交 | 编辑、查看驳回原因、提交审核（提交按钮文案为「重新提交」） |
| 已上线 | 审核通过，楼盘已上线 | 查看详情、编辑配置、下线 |
| 已下线 | 楼盘临时下线，可重新上线 | 查看详情、编辑配置、重新上线 |
