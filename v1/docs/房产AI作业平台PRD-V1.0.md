## 一、项目概述

### 1.1 项目背景

随着新房市场竞争加剧，经纪人获取楼盘信息、了解产品卖点、掌握销售话术的能力成为成交的关键。然而，传统模式下：

- **信息获取滞后**：经纪人依赖线下踩盘或转发资料，无法快速全面了解楼盘
- **能力参差不齐**：产品知识掌握程度不一，导致约看率、成交率不稳定
- **物料质量不一**：探盘成果标准不统一，生成的新媒体内容质量难以保障
- **管理抓手缺失**：管理层难以实时掌握经纪人任务执行进度，缺乏数据支撑

### 1.2 产品定义

新房AI作业流程是一款服务于新房业务的 SOP 系统，核心理念四个字：**探-练-推-约**

| 关键字   | 含义    | 具体动作                       |
| ----- | ----- | -------------------------- |
| **探** | 探盘任务  | 经纪人到楼盘现场，采集真实照片、视频、文字等物料   |
| **练** | 产品力练习 | 基于探盘成果进行 AI 话术训练，快速掌握楼盘卖点  |
| **推** | 新媒体获客 | AI 生成小红书、公众号等平台推广内容，线上引流获客 |
| **约** | 现场邀约  | 录音上传分析，优化话术，提升邀约率和成交率      |

> **产品 Slogan**：「探得真材，炼出专业，推向市场，约见成交」

平台通过标准化探盘任务、AI 物料生成、产品力训练三大核心能力，形成从「新盘入库 → 探盘执行 → 内容获客 → 能力提升」的完整闭环。

### 1.3 核心价值

#### 1.3.1 顶层价值

| 价值维度      | 具体描述                                   |
| --------- | -------------------------------------- |
| **新媒体获客** | 通过探盘获取真实物料，AI 生成小红书、公众号等平台高质量内容，实现线上获客 |
| **产品力提升** | 基于楼盘的话术训练，帮助经纪人快速掌握产品卖点，提高邀约率和成交率      |
| **管理可视化** | 实时追踪任务执行进度，数据看板辅助管理层科学决策，沉淀数据资产        |

#### 1.3.2 步骤价值

| 步骤    | 核心价值       | 详细说明                                  |
| ----- | ---------- | ------------------------------------- |
| **探** | 真实物料，标准统一  | 标准化探盘任务流程，确保各经纪人采集的物料质量统一、格式规范        |
| **练** | AI 训练，快速提升 | 基于探盘成果，自动关联训练题库，AI 实时反馈，加速经纪人掌握产品知识   |
| **推** | AI 生成，降本增效 | AI 自动生成小红书、公众号等平台推广内容，无需人工撰写，降低内容生产成本 |
| **约** | 录音复盘，持续优化  | 邀约录音 AI 分析，量化话术能力，给出改进建议，辅助经纪人持续提升    |

### 1.4 项目范围

#### 1.4.1 PC管理后台用户

| 用户角色         | 职责说明                              |
| ------------ | --------------------------------- |
| **新房项目维护人员** | 负责新房录入（需审核）、上线操作（无需审核）            |
| **产品运营人员**   | 负责 AI 话术训练题库准备、新房项目上线默认配置项、楼盘信息编辑 |
| **区域经理**     | 负责探盘任务下发、任务跟进追踪、数据分析；**不能**编辑楼盘信息 |

#### 1.4.2 小程序侧用户

| 用户角色     | 职责说明                                           |
| -------- | ---------------------------------------------- |
| **经纪人**  | 执行探盘任务（收集物料、生成内容、AI话术训练、录音分析）、观测自己的任务执行数据和数据看板 |
| **区域经理** | 移动办公，负责任务下发、跟进追踪、数据分析，可查看团队数据看板                |

**小程序双角色说明**：经纪人和区域经理共用一个小程序，通过权限控制功能可见性。

## 二、市场与竞品分析

### 2.1 目标市场

#### 战略定位

**新房新媒体获客作业标准定义者 + 能力中台**

本质：把经纪人作业能力从"个体经验"升级为"系统化输出"，解决规模化发展的核心瓶颈。

#### 目标客群

房地产经纪公司（含新房分销业务），优先选择：

- 公司规模：≥50人经纪人
- 业务特征：月均楼盘≥5个、新房占比≥30%
- 意愿度：有新媒体探索意愿

#### 目标用户画像

| 用户类型    | 占比  | 核心诉求    |
| ------- | --- | ------- |
| 新人经纪人   | 60% | 快速变专业   |
| 区域经理/店长 | 25% | 数据化追踪考核 |
| 高层管理者   | 15% | 策略决策有支撑 |

#### 扩展路径

| 阶段     | 内容           |
| ------ | ------------ |
| **短期** | 内部2～3个大区试点验证 |
| **中期** | 全国20+区域规模化复制 |
| **长期** | 外部市场生态拓展     |

### 2.2 目标用户

#### 核心用户（有痛点，需要产品解决）

| 用户角色        | 核心需求                  | 0-1阶段痛点                                                                            |
| ----------- | --------------------- | ---------------------------------------------------------------------------------- |
| **经纪人**     | 快速了解楼盘、标准化产出物料、提升话术能力 | 踩盘靠腿，信息靠记，笔记标准不统一；内容创作弱，不知从何下手；10分钟通关背不下来，人工考核有压力；不知道什么内容有效，缺乏反馈；话术改进靠经验，不知道具体哪里要改 |
| **区域经理/店长** | 掌握团队执行情况，数据化考核团队      | 任务追踪靠问，全靠嘴；产品通关需要现场组织，占用大量时间；经纪人完成质量参差不齐，难以量化；有任务但团队不积极，缺乏抓手；物料整合、模板评选工作量大         |
| **管理层**     | 数据看板、策略决策有依据          | 各系统数据分散，看不到全貌；不知道经纪人真实能力水平；不知道内容投放效果；产品力无法量化评估                                     |

#### 配合角色（职责说明）

| 用户角色         | 职责说明            |
| ------------ | --------------- |
| **新房项目维护人员** | 负责新房资料的录入和更新    |
| **产品运营人员**   | 配合维护新房资料和探盘任务管理 |

### 2.3 竞品分析

#### 传统SOP vs AI平台的替代关系

| 传统SOP环节       | 传统方式           | 现有替代方案         | AI平台差异化                                  |
| ------------- | -------------- | -------------- | ---------------------------------------- |
| **探（空看学习）**   | 经纪人自己踩盘、记笔记    | 线下踩盘，靠个人能力     | 标准化任务流程，统一采集规范，AI自动整理物料                  |
| **练（产品通关）**   | 靠背书、人考核、10分钟通关 | 人工培训，店长/区经现场考核 | AI对话练习+实时评分，无需人工考核                       |
| **推（朋友圈/集攻）** | 自己写朋友圈，发给店长汇总  | 个人能力参差不齐       | AI生成高质量文案，一键分发多平台                        |
| **约（带看签约）**   | 靠经验、靠感觉，不知道效果  | 经纪人自己判断        | **约前：AI话术训练提升邀约能力；约后：录音分析复盘总结**，数据看板追踪转化 |

#### 竞品类型

| 竞品类型       | 现状                 | 劣势             | 本产品差异化          |
| ---------- | ------------------ | -------------- | --------------- |
| **传统线下方式** | 经纪人自己踩盘、人工培训、人工写文案 | 效率低、质量不一、无量化反馈 | 标准化+AI自动化+数据量化  |
| **内部人工管理** | 店长/区经靠嘴问、靠眼看       | 无实时抓手、无法量化考核   | 数据看板+实时追踪+能力雷达图 |
| **分散系统**   | 各系统数据分散，不连通        | 数据孤岛，无法形成闭环    | 全流程数据打通，形成闭环    |

### 3.4 各模块规则

##### 新盘管理规则

| 规则类型     | 规则内容                                                     |
| -------- | -------------------------------------------------------- |
| **录入规则** | 按字段库完整录入，包含以下字段                                          |
| **审核规则** | **分阶段规划**：当前人工审核（奇点总部人员）→ 未来人工审核（新盘维护人员）或AI辅助审核          |
| **上线规则** | 上线后自动通知可下发人员、自动关联训练题库、上线时配置下线时间                          |
| **下线规则** | 下线前检查是否有正在执行的任务；下线时需通知任务下发人员、执行人员**通知方式**：公告 / 私信 / 任务通知 |
| **权限规则** | 新盘管理人员具有最高权限                                             |
| **触发动作** | 通知站内推送 + 小程序推送                                           |

## 五、功能模块详述

### 5.1 新盘管理

#### 5.1.1 功能概述

新盘管理是整个 SOP 的起点，负责新盘资料的录入、审核、新房项目的上下线管理。

**新盘生命周期流程**

```
录入 → 待审核 → 通过/已驳回 → 上线/下线
```

**角色与权限**

| 角色     | 权限        |
| ------ | --------- |
| 新房维护人员 | 录入、编辑     |
| 审核人员   | 审核通过/驳回   |
| 可下发人员  | 接收通知、下发任务 |

**新盘状态流转**

| 状态   | 说明             | 流转            |
| ---- | -------------- | ------------- |
| 草稿   | 初始录入状态，尚未提交审核  | → 待审核（提交审核）   |
| 待审核  | 提交等待审核         | → 审核通过 / 审核驳回 |
| 审核通过 | 楼盘信息已确认，待上线    | → 已上线（上线操作）   |
| 已驳回  | 审核不通过，需修改后重新提交 | → 待审核（重新提交审核） |
| 已上线  | 已上线，可下发探盘任务    | → 已下线（下线操作）   |
| 已下线  | 已下线，可重新上线      | → 已上线（上线操作）   |

**状态与操作关联表**

| 状态   | 查看 | 编辑 | 删除 | 提交审核 | 审核 | 上线 | 下线 |
| ---- | -- | -- | -- | ---- | -- | -- | -- |
| 草稿   | ✅  | ✅  | ✅  | ✅    | -  | -  | -  |
| 待审核  | ✅  | -  | -  | -    | ✅  | -  | -  |
| 审核通过 | ✅  | -  | ✅  | -    | -  | ✅  | -  |
| 已驳回  | ✅  | ✅  | ✅  | -    | -  | -  | -  |
| 已上线  | ✅  | -  | -  | -    | -  | -  | ✅  |
| 已下线  | ✅  | -  | ✅  | -    | -  | ✅  | -  |

**审核操作说明**：

- 待审核状态点击「审核」跳转到楼盘详情页进行审核
- 详情页审核区域包含：审核意见输入、通过/驳回按钮
- 已驳回状态：编辑后提交按钮显示为「重新提交」

### 5.2 探盘任务

#### 5.2.1 功能概述

探盘任务是获取楼盘真实物料的核心环节，标准化任务流程，确保物料质量。

**探盘任务包含三个子任务**：

- **探盘路书**（必选）：生成新媒体内容，详见A.新媒体内容生成
- **话术训练**（可选）：AI话术训练，详见5.3话术训练
- **邀约录音上传**（可选）：录音分析，详见5.4约看录音分析

**三者并行执行，互不依赖，各自独立完成。**

**可下发人员定义**：

- PC管理后台：配置了探盘任务下发权限的人
- 小程序侧：上线项目时选择的"可下发任务人"

**与5.1上线配置的关联**：任务下发时可按需调整5.1上线的配置（打卡范围、点位配置等）。

#### 5.2.2 任务类型详述

**A. 新媒体内容生成**

| 内容类型   | 说明             | 产出物            |
| ------ | -------------- | -------------- |
| 小红书文案  | 针对小红书平台风格的推广文案 | 500-800字图文脚本   |
| 小红书封面图 | 吸引眼球的封面设计      | 3:4竖图          |
| 公众号文案  | 专业深度的楼盘介绍      | 1000-1500字图文脚本 |
| 公众号图片  | 配合文案的配图        | 多张横图           |
| 短视频脚本  | 15-60秒口播脚本     | 分镜头脚本          |
| 短视频内容  | AI 生成口播文字 + 封面 | 可直接录制          |

**B. 产品力训练**

> 详见5.3话术训练模块，此处引用不做重复描述。

**C. 约看录音分析**

- **录音上传**：经纪人上传真实邀约录音
- **AI 分析**：通过提示词让大模型分析录音内容
- **分析维度**：话术技巧、客户需求挖掘、改进建议
- **客户关联**：经纪人可选择将分析结果关联到自己的客户

#### 5.2.3 探盘任务执行流程

**任务完成判定标准**：

- **探盘路书子任务**：文案确认后即完成（图片/视频为耗时任务，后台继续生成）
- **整个探盘任务**：探盘路书 + 话术训练 + 邀约录音上传**全部完成**才算完成
- **无需审核流程**：提交后直接进入下一状态

**执行流程（探盘路书）**：

```
阶段1：定位打卡
├── 到达楼盘现场
├── 系统定位校验（范围内才可打卡）
└── 确认到达

阶段2：点位采集
├── 按配置完成各点位资料采集
├── 采集方式：拍照/视频/录音/文字
└── AI 初审：照片清晰度、视频时长等自动检测

阶段3：文案确认
├── 经纪人选择需要生成的内容
├── AI 生成对应内容
└── 经纪人确认提交
    └── ⭐ 确认后探盘路书子任务完成（图片/视频后台异步生成）

阶段4：AI 内容生成（后台执行）
├── 生成各平台内容（可退出小程序）
├── 实时进度展示
├── 生成完成后推送通知
└── 经纪人确认成果
```

#### 5.2.4 任务可见范围配置

**功能说明**：配置哪些组织/人员可以看到该任务

**配置项**：

- 按组织架构选择：事业部 → 区域 → 大区 → 门店
- 按人员选择：指定经纪人
- 支持组合选择

**说明**：选择部门后，该部门及所有下级部门、人员自动纳入可见范围。

#### 5.2.5 异常处理说明

##### 打卡位置偏移

| 偏移范围     | 处理方式                      |
| -------- | ------------------------- |
| 200米以内   | ✅ 正常打卡通过                  |
| 200-500米 | ⚠️ 弹出警告，用户确认后可继续，记录"疑似偏移" |
| 500米以上   | 🚫 强制拦截，记录"打卡偏移异常"        |

**申诉机制**：用户可申诉 → 店长/管理员审批 → 24小时内处理

##### AI生成失败

| 场景     | 处理方式                                    |
| ------ | --------------------------------------- |
| 重试机制   | 自动重试3次，第1次失败→自动重试，第2次→询问用户，第3次→标记"生成异常" |
| 兜底方案   | 提供空白模板手动填写，避免任务卡死                       |
| 任务卡片展示 | 正常✅ / 生成中🔄 / 失败⚠️（可重试）                 |
| 人工介入   | 连续失败3次后可"联系客服"，1小时内完成                   |

##### 任务逾期

**配置时机**：

- 新房上线时配置：提醒时间 + 执行时间
- 任务下发时：直接带过来，可按需调整

**提醒机制**：

- 可配置多个提醒节点：轻度提醒 → 中度提醒 → 严重提醒 → 逾期
- 按优先级排序执行

**逾期规则**：

- 逾期后**不可完成任务**
- 任务直接变更为"已失效"

### 5.8 小程序内容日历

#### 5.8.1 功能概述

##### 5.8.1.1 背景与目标

房产经纪人日常需要接收并执行多种任务：探盘打卡、话术训练、文案撰写、视频制作等。当前各任务分散在不同入口，缺乏统一的通知和执行入口。

**本期目标：** 在小程序侧建立统一的任务消息通知和执行流程，覆盖探盘打卡、话术训练、文案任务、视频任务、自定义任务、待下发任务六种任务类型，支持项目制任务管理。

##### 5.8.1.2 核心场景

| 场景   | 描述                        |
| ---- | ------------------------- |
| 接收任务 | PC管理后台下发的任务自动推送到小程序       |
| 消息通知 | 日历页面顶部通知入口，聚合所有任务消息       |
| 执行任务 | 经纪人执行探盘打卡、话术训练、文案/视频制作等任务 |
| 进度追踪 | 下发人可跟进各阶段完成情况（后续规划）       |

##### 5.8.1.3 任务类型

| 类型    | 图标 | 说明              | 执行方式                |

##### 5.8.3.4 探盘任务执行流程

**四阶段说明：**

| 阶段  | 名称       | 必填 | 说明                                   |
| --- | -------- | -- | ------------------------------------ |
| 阶段1 | 打卡       | ✅  | 到达楼盘现场，定位签到                          |
| 阶段2 | 素材收集     | ✅  | 按配置的打卡点位完成资料采集（拍照/视频/录音/文本），必须完成才能提交 |
| 阶段3 | 文案/脚本生成  | -  | 后台生成文案/脚本，经纪人确认                      |
| 阶段4 | 封面图/视频生成 | -  | 后台基于确认内容生成封面图+短视频                    |

**⚠️ 后台执行说明（重要）**：

- 阶段3/4 生成过程中：
  - ✅ 用户可返回任务列表（内容日历页）
  - ✅ 用户可操作小程序其他页面
  - ✅ 用户可退出小程序
  - ✅ 后台服务持续运行，不受前端页面切换影响
- 任务卡片实时显示阶段状态（如"文案生成中 30%"、"视频生成中"）
- 生成完成后自动推送通知给用户

**打卡规则：**

- 必须先完成阶段1「定位打卡」，才能进入阶段2「点位打卡」
- 「必须打卡」的点位不可跳过；「可选打卡」的点位可跳过
- **完成规则**：
  - 「全部点位必须完成」：需完成所有必填点位才能进入阶段3
  - 「部分点位完成即可」：完成配置的必填点位的50%即可进入阶段3

**采集点位类型：**

| 类型 | 图标 | 说明     |
| -- | -- | ------ |
| 拍照 | 📷 | 拍照上传图片 |
| 视频 | 🎬 | 录制短视频  |
| 录音 | 🎤 | 录制语音   |
| 文本 | 📝 | 填写文字信息 |

### 5.10 小程序探盘任务执行

#### 5.10.1 功能概述

##### 5.10.1.1 背景

根据PC端探盘任务管理模块的设计，新盘上线后自动生成探盘任务，运营人员指定执行人员和通知人员后下发。本模块聚焦于小程序侧探盘任务的执行，为经纪人提供任务接收、执行、提交的全流程移动端体验。

##### 5.10.1.2 核心场景

| 场景   | 描述              |
| ---- | --------------- |
| 任务接收 | 收到探盘任务通知，查看任务详情 |
| 任务执行 | 按要求完成定位打卡、资料采集  |
| 结果提交 | 提交探盘资料，等待审核     |
| 任务追踪 | 查看任务进度和审核状态     |

##### 5.10.1.3 与PC端联动关系

```
PC管理后台
  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
  │  新盘管理    │──→│  探盘任务    │──→│  任务审核    │
  │  新盘上线    │    │  下发任务    │    │  审核结果    │
  └─────────────┘    └──────┬──────┘    └─────────────┘
                               ↓
小程序侧
  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
  │  任务通知    │──→│  任务执行    │──→│  结果提交    │
  │  接收推送    │    │  打卡采集    │    │  等待审核    │
  └─────────────┘    └─────────────┘    └─────────────┘
```

#### 5.10.2 任务通知

##### 5.10.2.1 通知来源

PC端下发探盘任务后，小程序收到推送。

##### 5.10.2.2 通知类型

| 类型    | 说明          |
| ----- | ----------- |
| 新任务下发 | 收到新的探盘任务    |
| 任务提醒  | 截止时间临近提醒    |
| 审核结果  | 探盘结果审核通过/驳回 |
| 催办通知  | 超时未完成被催办    |

#### 5.10.3 探盘任务卡片

##### 5.10.3.1 卡片字段

| 字段   | 说明                          |
| ---- | --------------------------- |
| 状态标签 | 待开始(灰)/进行中(黄)/已完成(绿)/已逾期(红) |
| 任务标题 | 楼盘名称+探盘                     |
| 进度信息 | 共X个阶段 · 已完成X个（与四阶段对应）       |
| 位置   | 📍 + 地址                     |
| 截止时间 | 🕐 + 时间，已逾期显示红色             |
| 右侧进度 | 圆形图标 + 百分比                  |
| 进度条  | 底部进度条，颜色随状态                 |

#### 5.10.4 探盘任务详情

##### 5.10.4.1 顶部信息区

- 楼盘名称
- 地址
- 任务状态
- 截止时间

##### 5.10.4.2 四阶段执行流程

| 阶段   | 名称     | 触发条件           | 说明                          |
| ---- | ------ | -------------- | --------------------------- |
| 阶段1  | 定位打卡   | 收到任务后点击「开始执行」  | 系统校验当前位置是否在楼盘定位范围内（打卡范围半径内） |
| 阶段2  | 点位打卡   | 阶段1完成后进入       | 按配置的打卡点位依次完成点位打卡，支持跳过可选点位   |
| 阶段3  | 文案确认   | 所有必填点位完成后进入    | 小程序端生成多平台文案和短视频脚本，用户确认或修改   |
| 阶段4a | 生成文案脚本 | 阶段3点击生成        | 生成多平台文案（小红书/抖音/朋友圈等）和短视频脚本  |
| 阶段4b | 生成封面视频 | 阶段4a完成后 + 用户确认 | 生成封面图和视频                    |
| 提交   | 提交审核   | 阶段4全部生成完成后     | 点击提交，任务状态变为「已提交」，等待PC端审核    |

**⚠️ 后台执行说明（重要）：**

- 阶段3/4a/4b 生成过程中：
  - ✅ 用户可返回任务列表（内容日历页）
  - ✅ 用户可操作小程序其他页面
  - ✅ 用户可退出小程序
  - ✅ 后台服务持续运行，不受前端页面切换影响
- 任务卡片实时显示阶段状态（如"文案生成中 30%"、"视频生成中"）
- 生成完成后自动推送通知给用户

**打卡规则：**

- 必须先完成阶段1「定位打卡」，才能进入阶段2「点位打卡」
- 「必须打卡」的点位不可跳过；「可选打卡」的点位可跳过
- **完成规则**：
  - 「全部点位必须完成」：需完成所有必填点位才能进入阶段3
  - 「部分点位完成即可」：完成配置的必填点位的50%即可进入阶段3

**采集点位类型（PC端配置）：**

| 类型 | 图标 | 说明     |
| -- | -- | ------ |
| 拍照 | 📷 | 拍照上传图片 |
| 视频 | 🎬 | 录制短视频  |
| 录音 | 🎤 | 录制语音   |
| 文本 | 📝 | 填写文字信息 |

#### 5.10.5 各阶段执行说明

##### 5.10.5.1 阶段1：定位打卡

**说明：** 到达楼盘现场后，点击"立即打卡"进行定位确认。

**交互：**

1. 点击「立即打卡」按钮
2. 系统获取当前位置
3. 判断是否在楼盘范围内（PC端配置打卡半径）
4. 签到成功，记录打卡时间

**页面内容：**

- 当前位置
- 距楼盘距离（在范围内✓）
- 打卡时间
- 立即打卡按钮

**状态：**

- 已打卡：显示打卡时间，不再显示按钮
- 未打卡：显示「立即打卡」按钮

##### 5.10.5.2 阶段2：采集执行

**说明：** 按PC端配置的打卡点位逐一完成资料采集。必须先完成阶段1「定位打卡」才能进入此阶段。

**交互：**

1. 查看打卡点位列表（按配置顺序展示）
2. 点击「打卡」进入单个点位采集
3. 根据点位配置的内容类型进行采集：
   - 拍照：调用相机拍摄（可设置最少数/最多数）
   - 视频：调用相机录制（可设置最短/最长时长）
   - 录音：调用麦克风录制
   - 文本：填写文字信息
4. 「必须打卡」的点位不可跳过；「可选打卡」的点位可跳过
5. 所有必填点位完成后自动进入阶段3

**单个点位打卡页面：**

- 点位要求：必须打卡/可选、采集内容类型、数量要求
- 拍照/视频/录音/文本采集区域
- 备注说明
- 跳过此点位 / 完成打卡按钮

**状态：**

- 未完成：显示添加按钮
- 进行中：显示当前数量/总数
- 已完成：显示绿色对勾

##### 5.10.5.3 阶段3：文案确认

**说明：** 完成点位采集后，小程序端自动生成多平台文案和短视频脚本，用户确认后进入下一阶段。

**⚠️ 支持后台执行：** 生成过程中用户可返回任务列表，后台持续运行

**页面状态：**

| 状态  | 显示内容            |
| --- | --------------- |
| 待生成 | 显示生成按钮          |
| 生成中 | 显示进度，可返回任务列表    |
| 待确认 | 显示生成的文案 + 确认按钮  |
| 已确认 | 显示"已确认"，自动进入阶段4 |

**交互：**

1. 点击「生成文案和脚本」
2. 查看生成进度（逐个显示生成状态）
3. 查看/编辑生成的文案和脚本
4. 点击确认，进入阶段4

##### 5.10.5.4 阶段4：成果生成

**说明：** 用户确认文案脚本后，小程序端生成封面图和视频。

**⚠️ 支持后台执行：** 生成过程中用户可返回任务列表，后台持续运行

**阶段4a - 生成封面图：**

- 点击生成封面图按钮
- 后台持续运行，可返回任务列表

**阶段4b - 生成视频：**

- 封面图生成完成后，点击生成视频
- 后台持续运行，可返回任务列表

**全部生成完成：**

- 显示生成内容列表（小红书文案/抖音文案/封面图/短视频）
- 点击提交审核

##### 5.10.5.5 提交审核

**条件：** 阶段1\~4全部完成

**按钮状态：**

- 不可提交：灰色按钮，显示"请完成必填项"
- 可提交：橙色主按钮，显示"提交审核"

**交互：**

1. 点击「提交审核」
2. 确认提交弹窗
3. 提交成功 → 任务状态变为「待审核」
4. 等待PC端审核人员审核

#### 5.10.6 任务卡片状态显示

**生成中状态（后台执行）：**

- 卡片底部显示当前生成状态
- 实时更新生成进度百分比
- 生成完成后自动更新为"✅ 生成完成"

#### 5.10.7 任务状态与流转

##### 5.10.7.1 小程序侧状态

| 状态  | 说明         | 可执行操作   |
| --- | ---------- | ------- |
| 待执行 | 收到任务未开始    | 开始执行    |
| 进行中 | 执行中，有阶段未完成 | 继续执行    |
| 已提交 | 已提交待审核     | 查看进度    |
| 已完成 | 审核通过       | 查看结果    |
| 已驳回 | 审核驳回需修改    | 修改后重新提交 |
| 已逾期 | 超时未完成      | 继续执行    |

##### 5.10.7.2 状态流转

```
待执行 ──→ 进行中 ──→ 已提交 ──→ 已完成
              ↓           ↓
           已逾期      已驳回 ──→ 进行中
```
