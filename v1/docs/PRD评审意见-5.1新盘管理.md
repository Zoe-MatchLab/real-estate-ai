## 二、数据结构与字段分析

### 2.1 新盘状态机

```
录入中 → 待审核 → 审核通过 → 已上线 ↔ 已下线
              ↘ 已驳回 →（修改后）→ 待审核
```

**状态分析：**

| 状态 | 是否可操作 | 可见范围 | 备注 |
|------|:---------:|---------|------|
| 录入中 | 编辑 | 仅录入人员可见 | 暂不可见，流程起点 |
| 待审核 | 审核通过/驳回 | 审核人员可见 | 审核人员可看到所有待审核楼盘 |
| 已驳回 | 修改后重新提交 | 录入人员 + 审核人员 | 修改操作权限归属未明确 |
| 审核通过 | 上线/下线 | 所有可下发人员 | 审核通过≠自动上线，需手动上线 |
| 已上线 | 下线 | 全体可见（假设） | 触发通知动作 |
| 已下线 | 重新上线 | — | 重新上线是否需要重新审核？未说明 |

**问题：**
- "审核通过"和"已上线"是两个独立动作——审核通过后楼盘处于什么状态？可见范围如何？
- "已下线"后重新上线，是否需要重新审核？流程未明确

**字段性质分类：**

| 性质 | 说明 | 字段示例 |
|------|------|---------|
| 静态字段 | 录入后基本不变 | 占地面积、建筑面积、容积率 |
| 动态字段 | 随时间变化需更新 | 均价、特价房、工程进展 |
| 关系字段 | 关联外部实体 | 最近地铁站名、竞品楼盘 |
| 描述字段 | 文本描述 | 园林特色、交通配套描述 |

**问题：**
1. **"基本信息必填"定义模糊**——5.1.4说"基本信息必填（81字段清单中的基本信息）"，但"基本信息"仅指字段清单第一部分12个字段，还是包括所有静态基础数据？
2. **动态字段更新机制缺失**——价格动态（首付、特价房、工程进展）是高频更新字段，谁来更新？提醒机制是什么？5.1.4只提到"项目政策过期前提醒"，未覆盖价格动态
3. **"编辑锁定"字段**——5.1.3提到"81字段区分必填/非必填/编辑锁定"，但字段清单中未标注哪些字段是"编辑锁定"
4. **字段冗余风险**——"环线位置"和"行政区"都属于位置信息，是否重复？"园林效果图"和"园林实景图"合并为一张图是否足够？
5. **"客群画像关键词"vs"客群标签"vs"标签"**——多个地方出现标签概念，是否共用同一套标签体系？
