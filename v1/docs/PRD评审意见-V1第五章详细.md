## 📌 各小节详细摘要

### 5.1 新盘管理

**核心功能点：**
- 新盘录入、审核、上下线全生命周期管理
- 81字段录入体系，区分必填/非必填/编辑锁定
- 新盘上线时自动配置探盘路书（打卡模版、点位、训练/录音规则）
- 触发通知可下发人员

**数据结构/字段：**
- 新盘状态：录入中 → 待审核 → 审核通过/已驳回 → 上线/下线
- 上线配置：楼盘信息、项目政策、可下发人员、通知人员
- 探盘路书配置：定位打卡、打卡点位、物料审核、文案/图片/视频生成、训练/录音
- PC端字段：81字段清单（基础信息、详细参数等）

**状态流转：**
```
录入 → 待审核 → 通过/已驳回 → 上线 ↔ 下线
              ↘ 修改后重新提交
```

**与其他模块的关系：**
- 上线后 → 自动通知可下发人员（5.2探盘任务）
- 上线后 → 自动关联训练题库（5.3话术训练）
- 上线后 → 可下发探盘任务（5.9小程序探盘任务下发）
- 探盘成果归档 → 物料库（5.5物料管理）

**可能存在的问题点：**
- 81字段清单具体内容未在PRD中详细列出，是否存在冗余字段？
- 审核驳回后修改流程的具体操作路径不够清晰
- 项目政策过期提醒与通知规则（7.2）之间的联动关系未明确
- 批量导入模板下载后的格式校验规则未定义

### 5.2 探盘任务

**核心功能点：**
- 探盘任务发起（PC端和小程序侧）
- 任务类型：新媒体内容生成（6种子类型）、产品力训练、约看录音分析
- 四阶段执行流程：定位打卡 → 点位采集 → 文案确认 → AI生成
- 任务可见范围配置（组织架构+人员）

**数据结构/字段：**
- 任务类型定义：内容平台、文案字数/格式要求
- 四阶段：打卡/采集/文案/成果生成
- PC端配置：打卡范围、完成规则、结果流向
- 任务状态：待分发 → 待执行 → 进行中 → 已提交 → 已完成/已驳回

**状态流转：**
```
待分发 → 待执行 → 进行中 → 已提交 → 已完成/已驳回
                          ↘ 已逾期
```

**与其他模块的关系：**
- 依赖5.1新盘上线，关联楼盘信息
- 探盘成果 → 5.5物料管理（物料库）
- 训练任务关联5.3话术训练
- 录音分析关联5.4约看录音分析
- 任务通知依赖7.2通知规则

**可能存在的问题点：**
- "约看录音分析"标注为"前台、后台暂未实现"，但在任务类型中被列出，可能造成混淆
- 四阶段中阶段3文案确认与阶段4成果生成的分工界面不够清晰（文案+脚本+封面图+视频全部放在阶段4？）
- AI生成内容的质量标准未定义
- 人工抽检比例的配置规则在5.2和7.1中均有提及但描述不完全一致

### 5.3 话术训练

**核心功能点：**
- AI对话式话术练习（经纪人扮演销售，AI扮演客户）
- 场景类型：产品介绍、邀约客户、首次带看、逼定、异议处理、竞品对比等
- 训练题库管理（人工创建，问答对格式）
- 能力评估：多维度评分 + 能力雷达图 + 个人能力阶梯分布
- PC端：题库管理、训练数据、能力看板

**数据结构/字段：**
- 场景类型清单（7种）
- 评估维度：产品熟悉度、话术逻辑、应变能力、情绪把控、其他
- 评分形式：评分制 + 总分制（ABCDE）
- 能力阶梯：单盘专家 → 商圈专家 → 行政区专家 → 全城专家

**状态流转：**
```
阅读资料 → 开始训练 → 训练完成 → 生成报告
```

**与其他模块的关系：**
- 新盘上线自动关联预设题库（5.1新盘管理）
- 训练入口在5.8小程序内容日历（训练任务卡片）
- 经纪人能力数据 → 5.6数据看板
- 客户画像打通暂不做（标记为后续规划）

**可能存在的问题点：**
- 题库来源仅标注"人工创建"，AI辅助生成能力未规划，会导致初期题库建设成本高
- 能力模型与业务价值（对获客转化的影响）未建立关联
- 成长路径"暂不做"，但个人能力阶梯分布已经设计，需确认两者关系
- 激励机制"暂不做"，但排行榜/勋章/积分等预期未管理，可能影响经纪人使用积极性

### 5.4 约看录音分析

**核心功能点：**
- 经纪人上传电话邀约录音（MP3/WAV）
- AI分析录音（24小时内返回）
- 分析维度：话术技巧、情绪判断、客户需求、改进建议
- 客户关联功能

**数据结构/字段：**
- 录音格式：MP3/WAV
- AI分析维度：话术技巧（开场白/需求挖掘/异议处理/促成技巧）、情绪判断（高/中/低意向度）、客户需求、改进建议
- 提示词可配置

**状态流转：**
```
上传录音 → 提交分析 → 后台处理 → 通知用户 → 查看结果
                ↘ 异常处理 → 重新分析/重新上传
```

**与其他模块的关系：**
- 录音来源：经纪人真实邀约电话
- 客户关联 → 5.13小程序客户管理
- 与5.2探盘任务中的"约看录音分析"类型的关系？标记"暂未实现"，但5.4也有相同说明，需统一

**可能存在的问题点：**
- 标注"暂未实现"，但PRD中花费较多篇幅描述设计，是否确认本期不实现？
- 24小时内返回分析结果，时间较长，是否有SLA保障？
- 录音文件大小限制未明确（只写"常规值限制"）
- 异常情况处理描述模糊（"引导用户重新分析或重新上传"）

### 5.9 小程序探盘任务下发

**核心功能点：**
- 移动端便捷下发探盘任务
- 简化下发流程（对比PC端完整表单）
- 新房上线通知 → 楼盘信息确认 → 配置任务 → 选择执行人员 → 确认下发
- 与PC端数据共用，同一任务表

**页面清单：**
- 新房上线通知（通知面板内）
- 楼盘信息确认页（楼盘详情）
- 配置探盘任务表单
- 执行人员选择页
- 确认下发弹窗

**任务配置表单字段：**
- 基础信息：任务名称（自动生成可改）、关联楼盘（自动带入）、楼盘地址（自动带入）、楼盘均价（自动带入）、优先级、截止时间
- 打卡模版配置：打卡范围（默认500米）、打卡规则、完成规则、结果流向（多选）
- 打卡点位配置：点位列表、新增/编辑/删除、开启状态、打卡内容类型、备注说明

**任务状态：**
```
待分发 → 待执行 → 进行中 → 已提交 → 已完成/已驳回
                          ↘ 已逾期（可继续执行）
```

**与其他模块的关系：**
- 依赖5.1新盘上线（只有上线的楼盘才能下发）
- 下发后 → 5.8内容日历通知执行人
- 与PC端5.2探盘任务共用数据

**可能存在的问题点：**
- 小程序下发和PC下发的"数据共用"——是否包括任务ID的共用？若是同一任务表，两个入口如何区分同一个任务？
- "打卡点位配置"说明"支持新增、编辑、删除"，但5.1上线配置时也有点位配置，两者的关系是什么？小程序端新增的点位是否会同步到楼盘配置？
- 人员选择"显示经纪人的当前任务数量"，这个数据从哪个接口获取？实时性如何保证？
- 状态"待分发"——小程序下发后状态直接变"待执行"，跳过了"待分发"状态，还是说小程序新建任务时也是"待分发"？

### 5.10 小程序探盘任务执行

**核心功能点：**
- 探盘任务四阶段执行：阶段1定位打卡、阶段2点位打卡、阶段3文案确认、阶段4成果生成
- 阶段3/4支持后台执行（退出小程序不受影响）
- 打卡规则：必须先阶段1才能进入阶段2；必填点位不可跳过；完成规则可配置
- 提交审核流程

**四阶段详细说明：**
| 阶段 | 名称 | 触发条件 | 说明 |
|------|------|---------|------|
| 阶段1 | 定位打卡 | 点击"开始执行" | 校验位置是否在楼盘打卡半径内 |
| 阶段2 | 点位打卡 | 阶段1完成 | 按配置点位采集，支持跳过可选点位 |
| 阶段3 | 文案确认 | 所有必填点位完成 | 生成多平台文案和脚本，用户确认 |
| 阶段4a | 生成封面图 | 阶段3确认 | 生成封面图 |
| 阶段4b | 生成视频 | 阶段4a完成 | 生成视频 |
| 提交 | 提交审核 | 阶段4全部完成 | 等待PC端审核 |

**数据结构：**
```javascript
{
  taskId, taskName, houseName, address, status,
  checkInConfig: { radius, rule, completionRule, timeLimit, resultFlow },
  stage: { current, stage1~stage4b },
  generationStatus: { stage3/stage4a/stage4b: { status, progress } },
  locationCheckIn: { status, time, location },
  checkPoints: [{ id, name, required, status, contentTypes }],
  publishTime, deadline, executor
}
```

**与其他模块的关系：**
- 承接5.9下发的探盘任务
- 生成内容归档至5.5物料管理
- 任务通知依赖7.2通知规则

**可能存在的问题点：**
- 阶段4在5.8中被描述为一个阶段，在5.10中被拆分为4a和4b，两处命名不一致
- 阶段3文案生成"小程序端自动生成"——小程序端如何调用AI生成能力？是否有后端API支撑？
- 阶段4生成封面图/视频，如果AI生成失败（比如上传了不合规内容），如何处理？有重试机制吗？
- "提交审核"后由PC端审核，但审核标准是什么？审核周期多久？超时未审核如何处理？
- 点位采集时视频的"最短/最长时长"配置项在5.10中提到但未详细说明
