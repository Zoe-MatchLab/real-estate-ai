### 异常处理建议（用户确认）

**配置时机**：
- 新房上线时配置：提醒时间 + 执行时间
- 任务下发时：直接带过来，**可按需调整**

**提醒时间配置**：
- 可配置**多个**提醒时间节点
- 按优先级排序：**轻度提醒** → **中度提醒** → **严重提醒** → **逾期**

**逾期规则**：
- **逾期后不可完成任务**
- 任务直接变更为"已失效"

#### 打卡位置偏移

| 偏移范围 | 处理方式 |
| --- | --- |
| 200米以内 | ✅ 正常打卡通过 |
| 200-500米 | ⚠️ 弹出警告，用户确认后可继续，记录"疑似偏移" |
| 500米以上 | 🚫 强制拦截，记录"打卡偏移异常" |
| 申诉机制 | 用户可申诉 → 店长/管理员审批 → 24小时内处理 |

#### AI生成失败

| 场景 | 处理方式 |
| --- | --- |
| 重试机制 | 自动重试3次，第1次失败→自动重试，第2次→询问用户，第3次→标记"生成异常" |
| 兜底方案 | 提供空白模板手动填写，避免任务卡死 |
| 任务卡片展示 | 正常✅ / 生成中🔄 / 失败⚠️（可重试） |
| 人工介入 | 连续失败3次后可"联系客服"，1小时内完成 |

#### 任务逾期

| 场景 | 处理方式 |
| --- | --- |
| 配置时机 | 新房上线时配置提醒时间+执行时间，任务下发时可调整 |
| 提醒机制 | 可配置多个提醒节点：轻度→中度→严重→逾期 |
| 逾期后 | **不可完成任务**，任务变更为"已失效" |
