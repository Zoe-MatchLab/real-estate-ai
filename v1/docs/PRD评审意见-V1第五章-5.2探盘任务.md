## 三、专项问题分析

### 3.1 问题1：状态字段缺失"已失效"

**问题描述：**

用户确认的任务状态为：待下发、进行中、**已失效**、已完成

但PRD 5.2章节中：

- 5.2 原文无独立的任务状态定义
- 5.9.5.1（小程序的PC端下发）定义了7种状态：`待分发、待执行、进行中、已提交、已完成、已驳回、已逾期`

缺失的状态：**"已失效"**

根据用户确认的异常处理规则：
> 逾期后**不可完成任务**，任务直接变更为"已失效"

**建议：**
1. 在5.2章节新增"任务状态"定义，明确包含"已失效"
2. 状态流转应补充：
   ```
   进行中 → 已失效（逾期后不可完成）
   ```
3. 与5.9.5.1的7种状态进行整合，统一状态定义

---

### 3.2 问题2：执行人定义方向相反【高优先级】

**问题描述：**

| 字段 | 用户定义 | PRD 5.2原文 |
| --- | --- | --- |
| 执行人 | 下发探盘任务的人 | 5.2.1定义可下发人员为"配置了探盘任务下发权限的人" |

PRD将"可下发任务的人"定义为执行人，但用户确认版中执行人应该是"下发探盘任务的人"。

**分析：**

这里存在**方向性歧义**：

- 如果"执行人"= 完成任务的人（经纪人）：则PRD定义相反
- 如果"执行人"= 下发任务的人（区域经理）：则定义符合，但名称不符

从业务逻辑分析：
- 任务列表中的"执行人"应该显示的是**执行该任务的经纪人**
- "下发人"才是下发任务的区域经理

**确认问题：**
> 请产品确认："执行人"字段的实际含义是"下发任务的人"还是"执行任务的人"？
> 如果是"下发任务的人"，建议字段改名为"下发人"以避免歧义。

---

### 3.3 问题3：完成进度字段完全缺失

**问题描述：**

用户确认的计算方式：
> 完成进度 = 有效期内已完成执行人 / 总执行人人数

PRD 5.2章节**完全没有**完成进度字段的定义或说明。

5.9.5.1的任务状态定义中也没有"完成进度"字段。

**缺失内容：**
1. 任务列表中"完成进度"列的展示形式（百分比？分数？）
2. "有效期内"的时间限定语没有在任何地方体现
3. "已完成执行人"的定义：什么算"已完成"？

**建议补充：**
- 在5.2.1或5.2.3中新增"完成进度"字段定义
- 说明计算逻辑：有效期内已完成执行的执行人数 / 总执行人数
- 明确"有效期内"的起止时间定义
- 明确"已完成"的判定标准（参照3个子任务全部完成）

---

### 3.4 问题4：提醒时间缺少"最多三个"的上限说明

**问题描述：**

用户确认：提醒时间**最多三个**，与下线配置有关

当前PRD 5.2.5异常处理中只提到：
> 可配置多个提醒节点：轻度提醒 → 中度提醒 → 严重提醒 → 逾期

没有提到**上限三个**的限制，也没有说明与"下线配置"的具体关联方式。

**建议补充：**
1. 明确提醒时间**上限为三个**
2. 说明提醒时间与5.1上线配置的关系：
   - 提醒时间可在新房上线时配置（引用5.1配置）
   - 任务下发时可按需调整
3. 补充提醒时间配置的具体示例：
   ```
   提醒1（轻度）：截止时间前24小时
   提醒2（中度）：截止时间前4小时
   提醒3（严重）：截止时间前1小时
   ```

---

### 3.5 问题5：操作按钮字段完全缺失【高优先级】

**问题描述：**

用户确认的操作：查看、下发、暂停、催办、编辑

PRD 5.2章节**完全没有**"操作"列的说明。

5.9.5.1中只提到了"下发、编辑、删除、取消、催办"，缺少**"暂停"**操作。

**操作按钮核对：**

| 操作 | 用户确认 | PRD 5.2 | PRD 5.9.5.1 | 状态 |
| --- | --- | --- | --- | --- |

| 下发 | ✅ | ❌ 无 | ✅ 有 | 已覆盖 |
| 暂停 | ✅ | ❌ 无 | ❌ 无 | **缺失** |
| 催办 | ✅ | ❌ 无 | ✅ 有 | 已覆盖 |
| 编辑 | ✅ | ❌ 无 | ✅ 有 | 已覆盖 |

**核心问题：**

1. **"暂停"操作完全缺失**：用户确认需要"暂停"操作，但PRD中没有
2. **"查看"操作缺失**：每个任务都应该可以查看详情
3. 操作按钮的**权限控制**完全没有说明：
   - 谁可以"下发"？只有下发人？
   - 谁可以"暂停"？下发人/管理员？
   - 谁可以"催办"？下发人/管理员？
   - 谁可以"编辑"？什么条件下可以编辑？

**建议：**
1. 在5.2章节新增"操作"列说明
2. 补充操作权限控制表：

| 操作 | 可执行角色 | 触发条件 | 状态影响 |

| 查看 | 所有人 | 无限制 | 无 |
| 下发 | 可下发人员 | 任务状态=待下发 | 状态→待执行 |
| 暂停 | 下发人/管理员 | 任务状态=进行中 | 状态→暂停中 |
| 催办 | 下发人/管理员 | 任务状态=进行中，临近截止 | 发送催办通知 |
| 编辑 | 下发人/管理员 | 任务状态=待下发/进行中 | 无 |
| 继续执行 | 下发人/管理员 | 任务状态=暂停中 | 状态→进行中 |
