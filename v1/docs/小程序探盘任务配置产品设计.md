## 一、功能定位

### 1.1 端侧分工

| 端 | 功能范围 |
|----|----------|
| **PC管理后台** | 任务创建、批量下发、人员配置、审核管理 |
| **小程序侧** | 任务配置与下发（执行人员）、任务执行（4阶段）、提交结果 |

### 1.2 小程序侧任务配置

当执行人员收到探盘任务后，可以：
1. **查看任务详情** - 楼盘信息、任务要求
2. **配置打卡设置** - 选择打卡点位、设置采集要求
3. **执行探盘任务** - 4阶段打卡流程
4. **提交探盘结果** - 提交审核

## 二、页面结构

### 2.1 页面清单

| 页面 | 路径 | 说明 |
|------|------|------|
| 探盘任务详情 | `task-detail.html` | 查看任务信息、配置打卡、执行 |
| 打卡配置 | `checkin-config.html` | 配置点位、设置采集要求 |

| 点位打卡 | `checkin-point.html` | 阶段2：点位采集 |
| 文案确认 | `copywriting-confirm.html` | 阶段3：AI文案确认 |
| 成果生成 | `results-generate.html` | 阶段4：查看生成成果 |
