## 一、产品概述

### 1.1 背景

根据PC端探盘任务管理模块的设计，新盘上线后自动生成探盘任务，运营人员指定执行人员和通知人员后下发。本模块聚焦于**小程序侧探盘任务的执行**，为经纪人提供任务接收、执行、提交的全流程移动端体验。

### 1.2 核心场景

| 场景 | 描述 |
|------|------|
| 任务接收 | 收到探盘任务通知，查看任务详情 |
| 任务执行 | 按要求完成定位打卡、资料采集 |
| 结果提交 | 提交探盘资料，等待审核 |
| 任务追踪 | 查看任务进度和审核状态 |

### 1.3 与PC端联动关系

```
┌─────────────────────────────────────────────────────────────┐
│                      PC管理后台                              │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐  │
│  │  新盘管理    │──→│  探盘任务    │──→│  任务审核    │  │
│  │  新盘上线    │    │  下发任务    │    │  审核结果    │  │
│  └─────────────┘    └──────┬──────┘    └─────────────┘  │
└─────────────────────────────┼───────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      小程序侧                              │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐  │
│  │  任务通知    │──→│  任务执行    │──→│  结果提交    │  │
│  │  接收推送    │    │  打卡采集    │    │  等待审核    │  │
│  └─────────────┘    └─────────────┘    └─────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

```
│  · 结果流向：多平台文案/视频脚本/封面   │
└─────────────────────────────────────────┘
```

**四阶段说明：**

| 阶段 | 名称 | 触发条件 | 说明 |
|------|------|----------|------|
| 阶段1 | 定位打卡 | 收到任务后点击「开始执行」 | 系统校验当前位置是否在楼盘定位范围内（打卡范围半径内） |
| 阶段2 | 点位打卡 | 阶段1完成后进入 | 按配置的打卡点位依次完成点位打卡，支持跳过可选点位 |
| 阶段3 | 文案确认 | 所有必填点位完成后进入 | 小程序端生成多平台文案和短视频脚本，用户确认或修改 |
| 阶段4a | 生成文案脚本 | 阶段3点击生成 | 生成多平台文案（小红书/抖音/朋友圈等）和短视频脚本 |
| 阶段4b | 生成封面视频 | 阶段4a完成后 + 用户确认 | 生成封面图和视频 |
| 提交 | 提交审核 | 阶段4全部生成完成后 | 点击提交，任务状态变为「已提交」，等待PC端审核 |

**⚠️ 后台执行说明（重要）**：
- 阶段3/4a/4b 生成过程中：
  - ✅ **用户可返回任务列表（内容日历页）**
  - ✅ 用户可操作小程序其他页面
  - ✅ 用户可退出小程序
  - ✅ **后台服务持续运行**，不受前端页面切换影响
- **任务卡片实时显示阶段状态**（如"文案生成中 30%"、"视频生成中"）
- 生成完成后**自动推送通知**给用户

**打卡规则：**
- 必须先完成阶段1「定位打卡」，才能进入阶段2「点位打卡」
- 「必须打卡」的点位不可跳过；「可选打卡」的点位可跳过
- **完成规则**：
  - 「全部点位必须完成」：需完成所有必填点位才能进入阶段3
  - 「部分点位完成即可」：完成配置的必填点位的50%即可进入阶段3

**结果流向配置（PC端配置）：**

| 配置项 | 阶段4生成内容 |
|--------|--------------|
| 多平台文案创作 | 适用于不同平台（抖音、小红书、微信公众号等）的探盘文案 |
| 视频脚本生成 | 探盘视频的完整脚本 |
| 封面内容生成 | 探盘报告的封面图设计 |
| 视频生成 | 自动合成的探盘讲盘视频 |

**采集点位类型（PC端配置）：**

| 类型 | 图标 | 说明 |
|------|------|------|
| 拍照 | 📷 | 拍照上传图片 |
| 视频 | 🎬 | 录制短视频 |
| 录音 | 🎤 | 录制语音 |
| 文本 | 📝 | 填写文字信息 |

### 4.4 阶段1：定位打卡

**说明：** 到达楼盘现场后，点击"立即打卡"进行定位确认。

**交互：**
1. 点击「立即打卡」按钮
2. 系统获取当前位置
3. 判断是否在楼盘范围内（PC端配置打卡半径）
4. 签到成功，记录打卡时间

**页面布局：**
```
┌─────────────────────────────────────────┐
│  【阶段1】✅ 定位打卡                    │
│                                         │
│  当前位置：上海市浦东新区张江路           │
│  距楼盘距离：156米 ✓（在范围内）         │
│                                         │
│  打卡时间：2026-03-31 10:30:00        │
│                                         │
│  [        立即打卡        ]              │
└─────────────────────────────────────────┘
```

**状态：**
- 已打卡：显示打卡时间，不再显示按钮
- 未打卡：显示「立即打卡」按钮

### 4.5 阶段2：采集执行

**说明：** 按PC端配置的打卡点位逐一完成资料采集。必须先完成阶段1「定位打卡」才能进入此阶段。

**交互：**
1. 查看打卡点位列表（按配置顺序展示）
2. 点击「打卡」进入单个点位采集
3. 根据点位配置的内容类型进行采集：
   - 拍照：调用相机拍摄（可设置最少数/最多数）
   - 视频：调用相机录制（可设置最短/最长时长）
   - 录音：调用麦克风录制
   - 文本：填写文字信息
4. 「必须打卡」的点位不可跳过；「可选打卡」的点位可跳过
5. 所有必填点位完成后自动进入阶段3

## 五、任务状态与流转

### 5.1 小程序侧状态

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| 待执行 | 收到任务未开始 | 开始执行 |
| 进行中 | 执行中，有阶段未完成 | 继续执行 |
| 已提交 | 已提交待审核 | 查看进度 |
| 已完成 | 审核通过 | 查看结果 |
| 已驳回 | 审核驳回需修改 | 修改后重新提交 |
| 已逾期 | 超时未完成 | 继续执行 |

### 5.2 状态流转

```
┌─────────────────────────────────────────────────────────────┐
│                    任务状态流转                             │
└─────────────────────────────────────────────────────────────┘

  待执行 ──→ 进行中 ──→ 已提交 ──→ 已完成
              ↓           ↓
           已逾期      已驳回 ──→ 进行中
```
