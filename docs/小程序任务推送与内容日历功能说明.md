# 小程序任务推送与内容日历功能说明文档
## 一、功能概述
实现PC管理后台新建探盘任务后，自动推送到小程序端，在内容日历中展示，支持任务搜索、查看、执行全流程。

## 二、功能原型页面
访问路径：`/miniprogram/calendar.html`
已实现效果：
1. ✅ 小程序标准5个Tab栏布局，内容日历为当前活跃Tab
2. ✅ 日历视图展示，支持日/周/月/列表四种视图切换
3. ✅ 顶部搜索栏支持任务名称/楼盘名称模糊搜索
4. ✅ PC端新任务下发时自动推送通知，日历对应日期显示红点标记
5. ✅ 今日任务列表展示，包含任务名称、截止时间、区域、状态等核心信息
6. ✅ 任务状态区分：待执行（蓝色）、进行中（橙色）、已完成（绿色）、已逾期（红色）

## 三、数据结构（PC-小程序同步字段）
PC端新建探盘任务后，同步到小程序的核心字段：
| 字段名 | 类型 | 说明 |
|-------|------|------|
| taskId | string | 任务唯一ID |
| taskName | string | 任务名称 |
| houseName | string | 关联楼盘名称 |
| area | string | 所属区域 |
| deadline | string | 截止时间（ISO格式） |
| status | string | 任务状态：pending（待执行）/progress（进行中）/completed（已完成）/timeout（已逾期） |
| taskDesc | string | 任务描述/要求 |
| points | array | 探盘点位列表 |
| publisher | string | 任务下发人 |
| publishTime | string | 下发时间 |

## 四、核心交互流程
### 1. PC端下发任务触发推送
1. 运营人员在PC管理后台「创建新房探盘任务」页面填写信息，选择执行人员，点击「保存并下发」
2. 后端自动推送任务到对应执行人员的小程序端
3. 小程序收到推送后：
   - 顶部弹出通知提示
   - 内容日历对应截止日期位置显示红点标记
   - 任务列表新增对应任务，状态为「待执行」

### 2. 小程序端任务操作
1. 用户点击推送通知/日历任务卡片进入任务详情页
2. 查看任务要求、点位配置、配套资料
3. 点击「开始执行」进入探盘采集流程
4. 完成所有采集项后提交任务，状态同步到PC端为「待审核」
5. PC端审核通过后，小程序任务状态更新为「已完成」

### 3. 搜索功能实现
支持两种搜索维度：
- **关键词搜索**：模糊匹配任务名称、楼盘名称，匹配结果高亮显示
- **时间筛选**：支持按日/周/月维度，或自定义起止时间筛选

## 五、一致性保证
### 1. 数据一致性
- 所有任务数据带版本号，每次更新版本号+1
- 小程序每5分钟自动同步一次最新任务列表，保证与PC端数据一致
- 离线操作数据本地缓存，网络恢复后自动同步

### 2. 状态一致性
任务状态流转完全同步：
- PC端修改任务 → 小程序实时更新
- 小程序提交任务 → PC端实时收到审核提醒
- 审核状态变更 → 小程序实时推送通知
