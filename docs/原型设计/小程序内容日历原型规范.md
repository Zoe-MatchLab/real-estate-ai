## 二、通知面板交互

### 2.1 触发方式

| 触发 | 动作 |
|------|------|
| 点击铃铛 🔔 | 底部面板向上滑出 |
| 点击遮罩 | 面板收回 |
| 点击 × 按钮 | 面板收回 |
| 下滑面板 | 面板收回 |

### 2.2 面板内容

```
┌─────────────────────────────────────────────────────────────┐
│  📋 通知                          [清空全部]          [×]  │
├─────────────────────────────────────────────────────────────┤
│  [全部] [探盘] [话术] [内容]                               │
├─────────────────────────────────────────────────────────────┤
│  🔴 中海汇德里探盘任务                                     │
│     2/4阶段进行中，请注意完成                    刚刚        │
├─────────────────────────────────────────────────────────────┤
│  ⚪ 万科城市之光话术训练                                    │
│     新任务已下发，请及时完成                  10分钟前       │
├─────────────────────────────────────────────────────────────┤
│  ⚪ 朋友圈文案生成                                        │
│     等待探盘完成后执行                          昨天        │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 通知项字段

| 字段 | 展示 | 说明 |
|------|------|------|
| 未读标记 | 🔴 红色圆点 | 未读状态 |
| 已读标记 | ⚪ 灰色圆点 | 已读状态 |
| 任务名称 | 14px深色粗体 | 任务标题 |
| 消息摘要 | 12px灰色 | 任务状态或提醒内容 |
| 发布时间 | 12px灰色 | 刚刚/10分钟前/昨天 |
| 跳转入口 | → | 点击进入详情 |

### 2.4 分类Tab

| Tab | 筛选内容 |
|-----|----------|
| 全部 | 所有通知 |
| 探盘 | type=exploration |
| 话术 | type=speech |
| 内容 | type=content |

## 四、任务类型定义

### 4.1 三种任务类型

| 类型 | 图标 | 阶段 | 执行入口 |
|------|------|------|----------|
| 探盘打卡 | 📍 | 4阶段（定位→点位→文案→成果）| task-detail.html |
| 话术训练 | 💬 | 无阶段，直接训练 | speech-train.html |
| 内容发布 | 📝 | 依赖探盘完成 | 提示弹窗 |

### 4.2 状态定义

| 状态 | 探盘 | 话术 | 内容 |
|------|------|------|------|
| 待开始 | 待开始 | 待执行 | 待解锁/待执行 |
| 进行中 | 进行中 | 进行中 | 进行中 |
| 已完成 | 已完成 | 已完成 | 已提交/已通过 |
| 已逾期 | 已逾期 | - | - |

## 五、跳转关系

```
calendar.html
    │
    ├── [🔔 通知铃铛] → 通知面板弹窗
    │                   │
    │                   ├── [探盘通知] → task-detail.html
    │                   ├── [话术通知] → speech-train.html
    │                   └── [内容通知] → 提示弹窗
    │
    ├── [探盘卡片] → task-detail.html（4阶段）
    │
    ├── [话术卡片] → speech-train.html
    │
    └── [内容卡片]
            ├── [已解锁] → 提示"功能即将上线"
            └── [未解锁] → 弹出提示"需先完成探盘打卡"
```

## 六、技术实现要点

### 6.1 CSS变量

```css
/* 任务类型颜色 */
--type-exploration: #FA8C16;   /* 探盘 - 橙色 */
--type-speech: #722ED1;        /* 话术 - 紫色 */
--type-content: #007AFF;       /* 内容 - 蓝色 */

/* 状态颜色 */
--status-pending: #86909C;      /* 待开始 - 灰色 */
--status-progress: #FF7D00;    /* 进行中 - 橙色 */
--status-completed: #00B42A;    /* 已完成 - 绿色 */
--status-overdue: #FA5151;      /* 已逾期 - 红色 */
```

## 七、验收标准

| # | 检查项 | 标准 |
|---|--------|------|
| 1 | 通知铃铛 | 显示未读数量红点 |
| 2 | 通知面板 | 底部弹窗正确弹出 |
| 3 | 分类筛选 | 全部/探盘/话术/内容 正确筛选 |
| 4 | 未读已读 | 红色/灰色圆点正确区分 |
| 5 | 清空全部 | 一键清除所有通知 |
| 6 | 探盘卡片 | 4种状态正确显示 |
| 7 | 话术卡片 | 3种状态 + 按钮正确 |
| 8 | 内容卡片 | 解锁状态判断正确 |
| 9 | 点击跳转 | 跳转到正确页面 |
| 10 | 内容锁定 | 未解锁时正确提示 |
