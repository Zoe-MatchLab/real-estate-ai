## 二、任务筛选设计

### 2.1 身份切换（考虑项）

| 方案 | 说明 | 适用场景 |
|------|------|----------|
| 需要身份切换 | 显示"经纪人" / "管理员" 切换入口 | 管理功能与个人任务分离 |
| 不需要切换 | 始终展示所有待下发任务 | 简化流程，快速作业 |

**推荐**：暂不设计身份切换，后续按需扩展。

### 2.2 时间维度

- **粒度**：按天展示
- **切换**：点击日历日期切换 / 左右滑动切换月份

### 2.3 任务类型Tab

| Tab | 图标 | 颜色 | 筛选 |
|-----|------|------|------|
| 全部 | - | 橙色 | 所有任务 |
| 探盘 | 📍 | #FA8C16 | type=house |
| 训练 | 📚 | #722ED1 | type=train |
| 文案 | ✏️ | #007AFF | type=text |
| 视频 | 🎬 | #00B42A | type=video |
| 自定义 | ⭐ | #FF7D00 | type=custom |
| 待下发 | ⏳ | #86909C | type=pending |

## 三、任务卡片详细设计

### 3.1 探盘任务卡片

```
┌─────────────────────────────────────────────────────────────┐
│  [📍探盘] [👤张三(发起人)]              ⏰ 今日18:00        │
│                                                             │
│  中海汇德里新盘探盘 (任务名称)                               │
│                                                             │
│  ┌─ 探盘进度 ──────────────────────────────┐               │
│  │  打卡    素材    文案/脚本   封面/视频   │               │
│  │  ████░░  ██░░░   ░░░░░     ░░░░░        │               │
│  └─────────────────────────────────────────┘               │
│                                                             │
│  ┌─ 训练进度 ──────────────────────────────┐               │
│  │  阅读    开始    完成                    │               │
│  │  ████░░  ██░░░   ░░░░░                  │               │
│  │                        [进入任务] [开始训练] │           │
│  └─────────────────────────────────────────┘               │
└─────────────────────────────────────────────────────────────┘
```

**字段说明**：

| 字段 | 位置 | 示例 |
|------|------|------|
| 任务类型图标 | 左上 | 📍 |
| 任务类型标签 | 图标后 | 探盘 |
| 任务发起人 | 左侧 | 👤张三 |
| 截止时间 | 右上 | ⏰ 今日18:00 |
| 任务名称 | 中部 | 中海汇德里新盘探盘 |
| 探盘进度条 | 4阶段横排 | 打卡/素材/文案/封面 |
| 训练进度条 | 3阶段横排 | 阅读/开始/完成 |
| 操作按钮 | 底部 | [进入任务] [开始训练] |

**阶段定义**：

| 阶段 | 探盘进度 | 训练进度 |
|------|----------|----------|
| 1 | 打卡 | 阅读资料 |
| 2 | 素材收集 | 开始训练 |
| 3 | 文案/脚本生成 | 训练完成 |
| 4 | 封面图/视频生成 | - |

**跳转逻辑**：
- `[进入任务]` → task-detail.html（对应当前阶段）
- `[开始训练]` → speech-train.html
