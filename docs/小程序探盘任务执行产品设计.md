## 一、产品概述

### 1.1 背景

根据PC端探盘任务管理模块的设计，新盘上线后自动生成探盘任务，运营人员指定执行人员和通知人员后下发。本模块聚焦于**小程序侧探盘任务的执行**，为经纪人提供任务接收、执行、提交的全流程移动端体验。

### 1.2 核心场景

| 场景 | 描述 |
|------|------|
| 任务接收 | 收到探盘任务通知，查看任务详情 |
| 任务执行 | 按要求完成定位打卡、资料采集 |
| 结果提交 | 提交探盘资料，等待审核 |
| 任务追踪 | 查看任务进度和审核状态 |

### 1.3 与PC端联动关系

```
┌─────────────────────────────────────────────────────────────┐
│                      PC管理后台                              │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐  │
│  │  新盘管理    │──→│  探盘任务    │──→│  任务审核    │  │
│  │  新盘上线    │    │  下发任务    │    │  审核结果    │  │
│  └─────────────┘    └──────┬──────┘    └─────────────┘  │
└─────────────────────────────┼───────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      小程序侧                              │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐  │
│  │  任务通知    │──→│  任务执行    │──→│  结果提交    │  │
│  │  接收推送    │    │  打卡采集    │    │  等待审核    │  │
│  └─────────────┘    └─────────────┘    └─────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

## 四、功能详述

### 4.1 任务通知

**来源：** PC端下发探盘任务后，小程序收到推送

**通知内容：**
```
┌─────────────────────────────────────┐
│ 🔴 中海汇德里探盘任务               │
│    您有新的探盘任务，请及时完成        │
│    刚刚                        →   │
└─────────────────────────────────────┘
```

**通知类型：**

| 类型 | 说明 |
|------|------|
| 新任务下发 | 收到新的探盘任务 |
| 任务提醒 | 截止时间临近提醒 |
| 审核结果 | 探盘结果审核通过/驳回 |
| 催办通知 | 超时未完成被催办 |

| 文本 | 📝 | 填写文字信息 |

### 4.4 阶段1：定位打卡

**说明：** 到达楼盘现场后，点击"立即打卡"进行定位确认。

**交互：**
1. 点击「立即打卡」按钮
2. 系统获取当前位置
3. 判断是否在楼盘范围内（PC端配置打卡半径）
4. 签到成功，记录打卡时间

**页面布局：**
```
┌─────────────────────────────────────────┐
│  【阶段1】✅ 定位打卡                    │
│                                         │
│  当前位置：上海市浦东新区张江路           │
│  距楼盘距离：156米 ✓（在范围内）         │
│                                         │
│  打卡时间：2026-03-31 10:30:00        │
│                                         │
│  [        立即打卡        ]              │
└─────────────────────────────────────────┘
```

**状态：**
- 已打卡：显示打卡时间，不再显示按钮
- 未打卡：显示「立即打卡」按钮

---

### 4.5 阶段2：采集执行

**说明：** 按PC端配置的点位要求，逐一完成资料采集。

**交互：**
1. 查看必填点位列表
2. 点击「添加」开始采集
3. 根据点位类型进行采集：
   - 拍照：调用相机拍摄
   - 视频：调用相机录制
   - 录音：调用麦克风录制
   - 文本：填写文字信息
4. 提交后显示采集数量

**页面布局：**
```
┌─────────────────────────────────────────┐
│  【阶段2】🔄 采集执行                    │
│                                         │
│  必填点位                               │
│  ┌─────────────────────────────────┐   │
│  │ 📷 沙盘照片            (0/1)      │   │
│  │    拍摄楼盘沙盘全景照片            │   │
│  │                    [添加]         │   │
│  └─────────────────────────────────┘   │
│  ┌─────────────────────────────────┐   │
│  │ 🎬 样板间视频          (0/1)      │   │
│  │    录制样板间讲解视频，不少于30秒  │   │
│  │                    [添加]         │   │
│  └─────────────────────────────────┘   │
│                                         │
│  选填点位                               │
│  ┌─────────────────────────────────┐   │
│  │ 📷 周边配套            (0/3)      │   │
│  │    拍摄周边配套设施照片           │   │
```

**状态：**
- 未完成：显示添加按钮
- 进行中：显示当前数量/总数
- 已完成：显示绿色对勾

### 4.8 提交审核

**条件：** 阶段1和阶段2全部完成（必填点位）

**按钮状态：**
- 不可提交：灰色按钮，显示"请完成必填项"
- 可提交：橙色主按钮，显示"提交审核"

**交互：**
1. 点击「提交审核」
2. 确认提交弹窗
3. 提交成功 → 任务状态变为「待审核」
4. 等待PC端审核人员审核

## 五、任务状态与流转

### 5.1 小程序侧状态

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| 待执行 | 收到任务未开始 | 开始执行 |
| 进行中 | 执行中，有阶段未完成 | 继续执行 |
| 已提交 | 已提交待审核 | 查看进度 |
| 已完成 | 审核通过 | 查看结果 |
| 已驳回 | 审核驳回需修改 | 修改后重新提交 |
| 已逾期 | 超时未完成 | 继续执行 |

### 5.2 状态流转

```
┌─────────────────────────────────────────────────────────────┐
│                    任务状态流转                             │
└─────────────────────────────────────────────────────────────┘

  待执行 ──→ 进行中 ──→ 已提交 ──→ 已完成
              ↓           ↓
           已逾期      已驳回 ──→ 进行中
```
