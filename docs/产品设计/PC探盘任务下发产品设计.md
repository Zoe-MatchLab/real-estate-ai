## 一、产品定位

### 1.1 功能定位

PC端探盘任务下发是**探盘任务管理模块**的核心功能，运营人员在PC端创建/配置探盘任务，指定执行人员和通知人员后下发，经纪人在小程序侧接收任务并执行。

### 1.2 端侧分工

| 端 | 功能范围 |
|----|----------|
| PC管理后台 | 任务创建、批量下发、批量导出、批量修改人员、审核管理；查看/编辑/下发系统自动生成的探盘任务（新盘上线触发） |
| 小程序侧 | 单条任务配置与下发（由执行人员或运营人员操作）、任务执行（4阶段）、提交结果 |

### 1.3 任务自动生成规则

- **触发时机**：新盘项目审核通过、自动上线后，系统自动生成待分发探盘任务
- **生成内容**：任务默认填充项目基础信息（楼盘名称、地址、均价、户型等）
- **后续流程**：运营人员收到通知 → 进入探盘任务管理页 → 编辑任务信息 → 指定执行/通知人员 → 下发

### 1.4 核心价值

| 价值 | 说明 |
|------|------|
| 标准化下发 | 运营人员批量选择楼盘、配置人员后一键下发 |
| 责任到人 | 明确执行人员，任务可追溯 |
| 实时同步 | 下发后小程序侧即时收到通知 |
| 阶段管控 | 4阶段执行流程，进度可视化 |

### 4.4 任务状态

| 状态 | 说明 | PC端可见操作 |
|------|------|-------------|
| 待分发 | 创建后未下发 | 下发、编辑、删除 |
| 待执行 | 已下发等待执行 | 查看详情、催办、取消 |
| 进行中 | 经纪人在执行中 | 查看进度、催办 |
| 已提交 | 经纪人已提交，等待审核 | 查看结果、审核 |
| 已完成 | 审核通过 | 查看结果 |
| 已驳回 | 审核驳回 | 重新下发 |
| 已逾期 | 超时未完成 | 查看进度、催办 |

**状态流转图：**
```
待分发 → 待执行 → 进行中 → 已提交 → 已完成/已驳回
                          ↘ 已逾期（可继续执行）
                          ↘ 已驳回（可重新下发→待执行）
```

**审核时机说明：**
- 执行人员在微信小程序完成所有阶段执行后，点击「提交」，任务状态变为「已提交」
- 只有「已提交」状态的任务，审核人员才能进行审核操作
- 审核通过 → 状态变为「已完成」
- 审核驳回 → 状态变为「已驳回」，执行人员修改后可重新提交

**逾期说明：**
- 超过截止时间未完成的任务，状态自动变为「已逾期」
- 已逾期任务仍可继续执行，执行后状态恢复为「进行中」

## 五、与小程序侧联动

### 5.1 联动流程

```
PC端                          小程序侧
──────────────────────────────────────────────────
创建任务
   ↓
选择楼盘 ──────────────────────────────────────
   ↓
配置人员 ──────────────────────────────────────
   ↓
点击「下发」 ──────→ 收到任务通知
   ↓                      ↓
状态变为「待执行」     进入任务详情
   ↓                      ↓
                   执行阶段1：定位打卡
   ↓                      ↓
                   执行阶段2：采集执行
   ↓                      ↓
                   执行阶段3：文案确认（后台生成）
   ↓                      ↓
                   执行阶段4：成果生成（后台生成）
   ↓                      ↓
提交审核 ←──────────────── 提交结果
   ↓
审核通过
   ↓                      ↓
状态变为「已完成」     收到审核通过通知
```

### 5.2 四阶段执行流程详解

执行人员在微信小程序侧完成探盘任务，分为以下4个阶段：

| 阶段 | 名称 | 触发条件 | 具体内容 | 状态显示 |
|------|------|----------|----------|----------|
| 阶段1 | 定位打卡 | 收到任务后点击「开始执行」 | 系统校验当前位置是否在楼盘定位范围内（打卡范围半径内） | 待打卡 → 打卡中 → 打卡成功 → 进入阶段2 |
| 阶段2 | 点位打卡 | 阶段1定位打卡完成后进入 | 按配置的打卡点位依次完成点位打卡（拍照/视频/录音/文本），支持跳过可选点位 | 点位1进行中 → 点位1完成 → 点位2... |
| 阶段3 | 文案确认 | 所有必填点位完成后进入 | AI根据采集资料生成讲解文案初稿，用户确认或修改 | 待确认 → 已确认 → 进入阶段4 |
| 阶段4 | 成果生成 | 阶段3确认后自动进入 | 根据「结果流向配置」自动生成：多平台文案/视频脚本/封面图/视频 | 生成中 → 已完成 |
| 提交 | 提交审核 | 阶段4完成后点击「提交」 | 所有阶段完成后，执行人员确认提交，状态变为「已提交」 | 已提交 |

**打卡规则说明：**
- 必须先完成阶段1「定位打卡」，才能进入阶段2「点位打卡」
- 点位打卡需按配置的顺序依次完成
- 「必须打卡」的点位不可跳过；「可选打卡」的点位可跳过
- 需在「点位打卡完成时间」内完成所有点位打卡，否则超时提醒

**完成规则：**
- 「全部点位必须完成」：所有配置的必填点位都完成才能进入阶段3
- 「部分点位完成即可」：完成配置的必填点位的50%即可进入阶段3

**结果流向配置：**
| 配置项 | 阶段4生成内容 |
|--------|--------------|
| 多平台文案创作 | 适用于不同平台（抖音、小红书、微信公众号等）的探盘文案 |
| 视频脚本生成 | 探盘视频的完整脚本，包含开场白、卖点讲解、结尾引导 |
| 封面内容生成 | 探盘报告的封面图设计 |
| 视频生成 | 自动合成的探盘讲盘视频（AI配音+素材合成） |

**进度计算逻辑：**
- 整体进度 = 已完成所有4个阶段的执行人数 / 总执行人数
- 单人进度 =（当前阶段序号 / 4）× 100%，精确到阶段内完成项百分比

### 5.3 通知机制

| 触发时机 | 通知对象 | 通知内容 |
|----------|----------|----------|
| 任务下发 | 执行人员 | 您有新的探盘任务 |
| 任务下发 | 通知人员 | 探盘任务已下发 |
| 截止提醒 | 执行人员 | 任务即将到期 |
| 超时催办 | 执行人员 | 任务已超时，请尽快完成 |
| 审核通过 | 执行/通知人员 | 探盘任务已完成 |
| 审核驳回 | 执行人员 | 探盘结果被驳回 |

### 9.2 重新下发交互

**触发：** 已驳回状态的任务点击「重新下发」

**交互流程：**
1. 点击「重新下发」按钮
2. 弹出下发弹窗，**默认填充原执行人员**（可修改）
3. 确认后任务状态变为「待执行」
4. 执行人员和通知人员收到任务通知
5. 驳回意见自动清空，记录重新下发操作日志

**弹窗样式：**
```

│  └─────────────────────────────────┘   │

```

### 9.3 删除前校验

**规则：**
- 仅管理员可删除任务
- 删除前系统校验：该任务是否已被小程序执行人员操作过（如已开始执行则不允许删除）
- 被引用任务（如已关联审核记录）不允许删除
- 删除前需二次确认
