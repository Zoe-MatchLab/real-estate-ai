# PRD 补充内容 - PM 视角 - 2.1 目标市场

> 来源视角：产品经理（PM）
> 补充章节：2.1 目标市场
> 编写日期：2026-04-10

---

## 背景说明

原始 PRD 中「2.1 目标市场」章节内容较为概括，缺少可落地的判断标准和优先级指引。以下从产品经理视角，对四个核心问题进行细化补充。

---

## 目标客群细化

**核心结论：产品最适合"已具备一定规模、新房业务占比高、正在寻求新媒体获客突破"的中大型经纪公司。**

具体来说，最需要这款产品的经纪公司通常具备以下特征：

| 特征维度 | 描述 |
|----------|------|
| **规模门槛** | 直营或合伙人门店 ≥ 10 家，经纪人 ≥ 50 人，区域跨度大 |
| **业务结构** | 新房分销业务占总营收 ≥ 30%，是公司的战略重点或核心增长引擎 |
| **管理痛点** | 经纪人能力参差不齐，内容产出质量不可控，管理层缺乏统一抓手 |
| **获客现状** | 依赖传统端口（58同城、安居客等）获客，新媒体（小红书、抖音、公众号）探索中但缺乏系统方法论 |
| **决策者画像** | 区域总监或运营负责人，对"数字化管理工具"有认知，有意愿投入 |

**不适合的客群**：
- 小型经纪公司（经纪人 < 20 人），体量不足以支撑 SOP 落地成本
- 以二手房为主业的公司，核心痛点与产品能力不匹配
- 公司决策链极长、IT 采购流程复杂的国有企业或上市中介

---

## 市场定位完善

**建议采用以下定位描述，替代原有表述：**

> **产品定位**：新房新媒体获客作业 SOP 系统
>
> **核心价值主张**：让每一位置业顾问都具备"专业探盘 + 高效获客"的双重能力，帮经纪公司把新房业务的获客效率提升一个量级。

**更清晰的四象限价值矩阵**：

| 对谁 | 解决什么问题 | 具体价值 |
|------|------------|----------|
| **经纪人** | 不会写、不知道怎么写、写出来没人看 | AI 自动生成高质量推广内容，探盘物料标准化 |
| **区域经理** | 任务下发后不知道进展，团队执行靠催 | 实时任务追踪，数据看板一目了然 |
| **管理层** | 不知道经纪人真实能力，决策缺数据 | 能力数据化，量化考核有依据 |
| **公司整体** | 新媒体获客做不起来，流量依赖外部端口 | 自建内容矩阵，降低获客成本，提升经纪人产品力 |

**一句话版本（对外 Pitch 用）**：
> 「用 AI 把探盘、练功、获客、邀约四个环节串成闭环，让经纪公司的新房业务从'靠经验'变成'靠系统'。」

## 目标规模细化

**如何定义"重点新房大区"——建议以下量化筛选标准：**

### 定量指标（初期试点筛选门槛）

| 指标 | 入选门槛 | 说明 |
|------|----------|------|
| 区域经纪人数量 | ≥ 50 人 | 样本足够大，能验证 SOP 效果 |
| 月均主推新房项目数 | ≥ 5 个 | 有足够多的楼盘供探盘任务执行 |
| 新房业务占比 | ≥ 30% | 大区核心业务与产品能力高度匹配 |
| 月均新房成交套数 | ≥ 30 套 | 业务活跃，工具落地有土壤 |
| 区域管理团队配置 | 有专职区域经理 | 有明确对接人，能推动落地 |

### 定性指标（加分项）

- 区域负责人对「数字化工具」接受度高，主动性强
- 该区域新媒体（如小红书）已有一定经纪人基础（≥ 30% 在做）
- 过往探盘任务有执行记录（哪怕是表格形式），说明有标准化需求

### 试点规模估算

以典型中大型经纪公司为例：

- 全国经纪人总数：1,000–5,000 人
- 重点新房大区数量：2–4 个（占总人数 20–40%）
- 试点覆盖经纪人：200–800 人
- 建议首批试点：选择 **1 个大区 + 1 个备用大区**，集中资源跑通闭环后复制

## 需求优先级

**初期（0-1 阶段）聚焦原则：「最痛、最有意愿、规模足够」**

### 优先级排序

| 优先级 | 客户类型 | 入选理由 |
|--------|----------|----------|
| **P0 - 首批试点** | 有明确新媒体获客需求 + 管理团队主动寻求工具 + 经纪人 ≥ 50 人的新房大区 | 痛点真实，决策链短，落地成功率高，是最佳验证场景 |
| **P1 - 第二批扩展** | 新房业务占比高但新媒体意识弱的大区 | 需要一定的用户教育，但如果 P0 跑出数据，证明价值后这类区域跟进意愿强 |
| **P2 - 后续复制** | 中小型区域或二手房为主的区域 | 需求相对分散，需要更多定制化，属于增长阶段目标 |

### 初期聚焦的三个「必须」

1. **必须是新房核心业务区**：确保产品解决的问题是该大区的核心痛点，而不是边缘需求
2. **必须有区域管理层支持**：工具落地需要自上而下推动，没有区域负责人背书，经纪人执行率难以保障
3. **必须有足够的项目数量支撑任务闭环**：少于 5 个在推楼盘，无法验证「探-练-推-约」完整闭环价值

### 需求验证优先级（功能层面）

基于目标客群痛点，建议首期功能优先级为：

| 优先级 | 功能模块 | 理由 |
|--------|----------|------|
| **P0** | 探盘任务执行 + 物料采集 | 切核心痛点，落地门槛低，成果可见 |
| **P0** | AI 内容生成（小红书/朋友圈） | 新媒体获客的核心卖点，差异化强 |
| **P1** | AI 话术训练（练） | 提升产品力，但需要楼盘内容先积累 |
| **P1** | 邀约录音分析（约） | 价值明确，但需要经纪人建立使用习惯 |
| **P2** | 数据看板 / 管理驾驶舱 | 管理层需求，但需要前期数据积累 |

---

## 总结

从 PM 视角，2.1 目标市场章节最需要补充的是**「谁、怎么选、优先谁」**三个判断标准。本补充内容提供了：

1. **目标客群细化**：中大型、新房为主、正在做/想做新媒体获客的经纪公司
2. **市场定位完善**：从「作业标准」升级为「新房新媒体获客 SOP 系统 + 能力提升闭环」
3. **目标规模细化**：5 项定量指标 + 2 项定性指标定义「重点新房大区」
4. **需求优先级**：P0/P1/P2 分级，首批聚焦痛点真实、意愿强、规模够的大区

以上供 PRD 评审参考，建议在正式 PRD 中以「试点大区筛选标准」和「功能优先级矩阵」的形式纳入正文。
