## 一、产品背景

### 1.1 问题现状

当前系统中涉及"谈判"的场景：
- **话术训练**：议价谈判模拟训练（培训场景）
- **新盘管理**：提到"谈判成功的任务可创建为新盘"

**问题**：
- 谈判任务是独立流程，与探盘任务、数据报表无联动
- 缺少统一的谈判任务配置和管理页面
- 无法在执行探盘任务时配置谈判目标

### 1.2 目标

设计一个小程序侧的**谈判任务配置页面**，让经纪人在探盘过程中或探盘完成后，可以：
1. 配置谈判目标（价格、优惠、配套等）
2. 设置谈判策略和话术
3. 记录谈判过程和结果
4. 与探盘报告关联

## 二、核心功能

### 2.1 功能清单

| 功能模块 | 功能点 | 说明 |
|----------|--------|------|
| **谈判任务创建** | 关联探盘任务 | 选择已完成的探盘任务 |
| | 填写谈判目标 | 预期价格、优惠条件、配套要求 |
| | 选择谈判类型 | 价格谈判、优惠谈判、配套谈判、综合谈判 |
| **谈判配置** | 谈判策略 | 锚定策略、让步策略、附加价值策略 |
| | 心理价位 | 底价、目标价、理想价 |
| | 关键卖点 | 项目核心卖点排序 |
| **谈判话术** | 参考话术 | AI推荐话术库 |
| | 自定义话术 | 经纪人自定义话术 |
| **谈判记录** | 谈判过程 | 记录每次谈判内容 |
| | 结果录入 | 谈判成功/失败/继续跟进 |
| **数据同步** | 关联报表 | 谈判数据同步到个人数据中心 |

## 五、谈判话术配置

### 5.1 话术类型

| 类型 | 说明 | 示例场景 |
|------|------|----------|
| 开场话术 | 建立信任，引入话题 | "王先生您好，感谢您抽出时间..." |
| 探需话术 | 了解客户真实需求 | "您买房最看重哪些方面呢？" |
| 呈现话术 | 展示房源优势 | "这个楼盘是区域内唯一..." |
| 处理异议话术 | 处理价格等异议 | "您的顾虑我理解，我帮您算一下..." |
| 逼定话术 | 促进成交决定 | "这个价格是今天专属的优惠..." |
| 促成话术 | 促成签约 | "您方便今天定下来吗？" |

### 5.2 话术库结构

```javascript
{
  scriptId: "SCRIPT_xxx",
  scriptName: "价格谈判话术",
  category: "negotiation",  // negotiation/sales/objection/closing
  tags: ["价格", "优惠", "常见问题"],
  scripts: [
    {
      type: "开场话术",
      content: "王先生您好，感谢您对我们项目的关注。我是您的专属置业顾问...",
     适用场景: "首次接触客户"
    },
    {
      type: "探需话术",
      content: "您买房最看重的是投资潜力还是自住舒适性呢？",
      适用场景: "了解需求"
    }
  ]
}
```

## 七、UI/UX 设计规范

### 7.1 视觉规范

| 元素 | 规范 |
|------|------|
| 主色调 | `#FA8C16` (品牌橙) |
| 状态色 | 进行中=蓝, 待谈判=橙, 已成交=绿, 已失败=红 |
| 卡片样式 | 白色背景, 12px圆角, 轻微阴影 |
| 间距 | 16px 统一间距 |
| 字体 | 系统字体, 标题15px, 正文14px, 辅助12px |

### 7.2 交互规范

| 规范 | 说明 |
|------|------|
| 必填标识 | 字段后加 `*` |
| 输入反馈 | 实时校验, 错误时字段变红 |
| 保存提示 | 草稿自动保存, 成功时 Toast 提示 |
| 删除确认 | 二次确认弹窗 |

### 7.3 可访问性

- 所有按钮有 `:focus-visible` 样式
- 表单字段关联 `label`
- 图片有 `alt` 文本
- 颜色对比度符合 WCAG 2.1 AA 标准
