## 一、项目概述

### 1.1 项目背景

随着新房市场竞争加剧，经纪人获取楼盘信息、了解产品卖点、掌握销售话术的能力成为成交的关键。然而，传统模式下：

- **信息获取滞后**：经纪人依赖线下踩盘或转发资料，无法快速全面了解楼盘
- **能力参差不齐**：产品知识掌握程度不一，导致约看率、成交率不稳定
- **物料质量不一**：探盘成果标准不统一，生成的新媒体内容质量难以保障
- **管理抓手缺失**：管理层难以实时掌握经纪人任务执行进度，缺乏数据支撑

### 1.2 产品定义

新房AI作业流程是一款服务于新房业务的 SOP 系统，核心理念四个字：**探-练-推-约**

| 关键字   | 含义    | 具体动作                       |
| ----- | ----- | -------------------------- |
| **探** | 探盘任务  | 经纪人到楼盘现场，采集真实照片、视频、文字等物料   |
| **练** | 产品力练习 | 基于探盘成果进行 AI 话术训练，快速掌握楼盘卖点  |
| **推** | 新媒体获客 | AI 生成小红书、公众号等平台推广内容，线上引流获客 |
| **约** | 现场邀约  | 录音上传分析，优化话术，提升邀约率和成交率      |

> **产品 Slogan**：「探得真材，炼出专业，推向市场，约见成交」

平台通过标准化探盘任务、AI 物料生成、产品力训练三大核心能力，形成从「新盘入库 → 探盘执行 → 内容获客 → 能力提升」的完整闭环。

### 1.3 核心价值

#### 1.3.1 顶层价值

| 价值维度      | 具体描述                                   |
| --------- | -------------------------------------- |
| **新媒体获客** | 通过探盘获取真实物料，AI 生成小红书、公众号等平台高质量内容，实现线上获客 |
| **产品力提升** | 基于楼盘的话术训练，帮助经纪人快速掌握产品卖点，提高邀约率和成交率      |
| **管理可视化** | 实时追踪任务执行进度，数据看板辅助管理层科学决策，沉淀数据资产        |

#### 1.3.2 步骤价值

| 步骤    | 核心价值       | 详细说明                                  |
| ----- | ---------- | ------------------------------------- |
| **探** | 真实物料，标准统一  | 标准化探盘任务流程，确保各经纪人采集的物料质量统一、格式规范        |
| **练** | AI 训练，快速提升 | 基于探盘成果，自动关联训练题库，AI 实时反馈，加速经纪人掌握产品知识   |
| **推** | AI 生成，降本增效 | AI 自动生成小红书、公众号等平台推广内容，无需人工撰写，降低内容生产成本 |
| **约** | 录音复盘，持续优化  | 邀约录音 AI 分析，量化话术能力，给出改进建议，辅助经纪人持续提升    |

### 1.4 项目范围

#### 1.4.1 PC管理后台用户

| 用户角色         | 职责说明                              |
| ------------ | --------------------------------- |
| **新房项目维护人员** | 负责新房录入（需审核）、上线操作（无需审核）            |
| **产品运营人员**   | 负责 AI 话术训练题库准备、新房项目上线默认配置项、楼盘信息编辑 |
| **区域经理**     | 负责探盘任务下发、任务跟进追踪、数据分析；**不能**编辑楼盘信息 |

#### 1.4.2 小程序侧用户

| 用户角色     | 职责说明                                           |
| -------- | ---------------------------------------------- |
| **经纪人**  | 执行探盘任务（收集物料、生成内容、AI话术训练、录音分析）、观测自己的任务执行数据和数据看板 |
| **区域经理** | 移动办公，负责任务下发、跟进追踪、数据分析，可查看团队数据看板                |

**小程序双角色说明**：经纪人和区域经理共用一个小程序，通过权限控制功能可见性。

## 二、市场与竞品分析

### 2.1 目标市场

#### 战略定位

**新房新媒体获客作业标准定义者 + 能力中台**

本质：把经纪人作业能力从"个体经验"升级为"系统化输出"，解决规模化发展的核心瓶颈。

#### 目标客群

房地产经纪公司（含新房分销业务），优先选择：

- 公司规模：≥50人经纪人
- 业务特征：月均楼盘≥5个、新房占比≥30%
- 意愿度：有新媒体探索意愿

#### 目标用户画像

| 用户类型    | 占比  | 核心诉求    |
| ------- | --- | ------- |
| 新人经纪人   | 60% | 快速变专业   |
| 区域经理/店长 | 25% | 数据化追踪考核 |
| 高层管理者   | 15% | 策略决策有支撑 |

#### 扩展路径

| 阶段     | 内容           |
| ------ | ------------ |
| **短期** | 内部2～3个大区试点验证 |
| **中期** | 全国20+区域规模化复制 |
| **长期** | 外部市场生态拓展     |

### 2.2 目标用户

#### 核心用户（有痛点，需要产品解决）

| 用户角色        | 核心需求                  | 0-1阶段痛点                                                                            |
| ----------- | --------------------- | ---------------------------------------------------------------------------------- |
| **经纪人**     | 快速了解楼盘、标准化产出物料、提升话术能力 | 踩盘靠腿，信息靠记，笔记标准不统一；内容创作弱，不知从何下手；10分钟通关背不下来，人工考核有压力；不知道什么内容有效，缺乏反馈；话术改进靠经验，不知道具体哪里要改 |
| **区域经理/店长** | 掌握团队执行情况，数据化考核团队      | 任务追踪靠问，全靠嘴；产品通关需要现场组织，占用大量时间；经纪人完成质量参差不齐，难以量化；有任务但团队不积极，缺乏抓手；物料整合、模板评选工作量大         |
| **管理层**     | 数据看板、策略决策有依据          | 各系统数据分散，看不到全貌；不知道经纪人真实能力水平；不知道内容投放效果；产品力无法量化评估                                     |

#### 配合角色（职责说明）

| 用户角色         | 职责说明            |
| ------------ | --------------- |
| **新房项目维护人员** | 负责新房资料的录入和更新    |
| **产品运营人员**   | 配合维护新房资料和探盘任务管理 |

### 2.3 竞品分析

#### 传统SOP vs AI平台的替代关系

| 传统SOP环节       | 传统方式           | 现有替代方案         | AI平台差异化                                  |
| ------------- | -------------- | -------------- | ---------------------------------------- |
| **探（空看学习）**   | 经纪人自己踩盘、记笔记    | 线下踩盘，靠个人能力     | 标准化任务流程，统一采集规范，AI自动整理物料                  |
| **练（产品通关）**   | 靠背书、人考核、10分钟通关 | 人工培训，店长/区经现场考核 | AI对话练习+实时评分，无需人工考核                       |
| **推（朋友圈/集攻）** | 自己写朋友圈，发给店长汇总  | 个人能力参差不齐       | AI生成高质量文案，一键分发多平台                        |
| **约（带看签约）**   | 靠经验、靠感觉，不知道效果  | 经纪人自己判断        | **约前：AI话术训练提升邀约能力；约后：录音分析复盘总结**，数据看板追踪转化 |

#### 竞品类型

| 竞品类型       | 现状                 | 劣势             | 本产品差异化          |
| ---------- | ------------------ | -------------- | --------------- |
| **传统线下方式** | 经纪人自己踩盘、人工培训、人工写文案 | 效率低、质量不一、无量化反馈 | 标准化+AI自动化+数据量化  |
| **内部人工管理** | 店长/区经靠嘴问、靠眼看       | 无实时抓手、无法量化考核   | 数据看板+实时追踪+能力雷达图 |
| **分散系统**   | 各系统数据分散，不连通        | 数据孤岛，无法形成闭环    | 全流程数据打通，形成闭环    |

### 3.4 各模块规则

##### 新盘管理规则

| 规则类型 | 规则内容 |
|----------|----------|
| **录入规则** | 按字段库完整录入，包含以下字段 |
| **审核规则** | **分阶段规划**：当前人工审核（奇点总部人员）→ 未来人工审核（新盘维护人员）或AI辅助审核 |

| **权限规则** | 新盘管理人员具有最高权限 |
| **触发动作** | 通知站内推送 + 小程序推送 |

### 5.2 探盘任务

#### 5.2.1 功能概述

探盘任务是获取楼盘真实物料的核心环节，标准化任务流程，确保物料质量。

#### 5.2.2 任务类型详述

**A. 新媒体内容生成**

| 内容类型   | 说明             | 产出物            |
| ------ | -------------- | -------------- |
| 小红书文案  | 针对小红书平台风格的推广文案 | 500-800字图文脚本   |
| 小红书封面图 | 吸引眼球的封面设计      | 3:4竖图          |
| 公众号文案  | 专业深度的楼盘介绍      | 1000-1500字图文脚本 |
| 公众号图片  | 配合文案的配图        | 多张横图           |
| 短视频脚本  | 15-60秒口播脚本     | 分镜头脚本          |
| 短视频内容  | AI 生成口播文字 + 封面 | 可直接录制          |

**B. 产品力训练**

- **训练题库关联**：新盘上线时自动关联预设题库
- **题库来源**：基于楼盘基础信息自动生成 + 人工补充
- **训练形式**：AI 对话式练习，模拟客户问答

**C. 约看录音分析**（暂未实现）

- **录音上传**：经纪人上传真实邀约录音
- **AI 分析**：通过提示词让大模型分析录音内容
- **分析维度**：话术技巧、客户需求挖掘、改进建议
- **客户关联**：经纪人可选择将分析结果关联到自己的客户

#### 5.2.3 探盘任务执行流程

```
阶段1：定位打卡
├── 到达楼盘现场
├── 系统定位校验（范围内才可打卡）
└── 确认到达

阶段2：点位采集
├── 按配置完成各点位资料采集
├── 采集方式：拍照/视频/录音/文字
├── AI 初审：照片清晰度、视频时长等自动检测
└── 人工抽检：按配置比例随机抽检

阶段3：文案确认
├── AI 生成多版本文案
├── 经纪人选择/修改
└── 确认提交

阶段4：AI 内容生成
├── 生成各平台内容（后台执行，可退出小程序）
├── 实时进度展示
├── 生成完成后推送通知
└── 经纪人确认成果

提交审核
├── 提交到 PC 端审核
├── 通过：状态变为「已完成」，同步到物料库
└── 驳回：修改后重新提交
```

#### 5.2.4 任务可见范围配置

**功能说明**：配置哪些组织/人员可以看到该任务

**配置项**：

- 按组织架构选择：事业部 → 区域 → 大区 → 门店
- 按人员选择：指定经纪人
- 支持组合选择

**说明**：选择部门后，该部门及所有下级部门、人员自动纳入可见范围。

## 六、非功能性需求

### 6.1 性能要求

| 指标     | 要求                           |
| ------ | ---------------------------- |
| 页面加载   | 首屏 < 2秒                      |
| API 响应 | < 500ms（普通接口），< 3秒（AI 生成类接口） |
| 并发支持   | 支持 1000+ 同时在线                |

### 6.2 安全要求

- 用户认证：Token 鉴权，支持 refresh token
- 权限控制：接口级权限校验，防止越权操作
- 数据安全：敏感数据加密传输，录音文件加密存储

### 6.3 兼容性要求

| 端       | 兼容要求                                       |
| ------- | ------------------------------------------ |
| PC 管理后台 | Chrome 90+、Firefox 88+、Safari 14+、Edge 90+ |
| 小程序     | 微信小程序 2.20+，支持 iOS 12+、Android 8+          |

## 八、风险与依赖

### 8.1 技术风险

| 风险         | 影响        | 应对措施          |
| ---------- | --------- | ------------- |
| AI 生成质量不稳定 | 内容可能不符合要求 | 人工审核 + AI 抽检  |
| 定位精度不足     | 打卡位置偏移    | 放宽打卡范围 + 人工确认 |
| 录音转写准确率    | 影响分析结果    | 人工复核入口        |

### 8.2 业务风险

| 风险      | 影响       | 应对措施        |
| ------- | -------- | ----------- |
| 经纪人配合度低 | 任务执行率低   | 绩效激励 + 店长督进 |
| 楼盘信息滞后  | 内容过时     | 定期更新楼盘资料    |
| 数据隐私    | 录音包含客户信息 | 加密存储 + 授权访问 |

### 8.3 外部依赖

| 依赖项    | 说明        | 接入方式     |
| ------ | --------- | -------- |
| 微信小程序  | 承载小程序端    | 微信开放平台   |
| AI 大模型 | 内容生成、话术分析 | 第三方 API  |
| 地图服务   | 定位打卡      | 腾讯地图 API |
| 推送服务   | 小程序通知     | 微信订阅消息   |
