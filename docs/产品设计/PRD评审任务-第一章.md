## 评审内容

### 1.1 项目背景

随着新房市场竞争加剧，经纪人获取楼盘信息、了解产品卖点、掌握销售话术的能力成为成交的关键。然而，传统模式下：

- **信息获取滞后**：经纪人依赖线下踩盘或转发资料，无法快速全面了解楼盘
- **能力参差不齐**：产品知识掌握程度不一，导致约看率、成交率不稳定
- **物料质量不一**：探盘成果标准不统一，生成的新媒体内容质量难以保障
- **管理抓手缺失**：管理层难以实时掌握经纪人任务执行进度，缺乏数据支撑

### 1.2 产品定义

房产AI作业平台是一款服务于新房业务的 SOP 系统，核心理念四个字：**探-练-推-约**

| 关键字 | 含义 | 具体动作 |
|--------|------|----------|
| **探** | 探盘任务 | 经纪人到楼盘现场，采集真实照片、视频文字等物料 |
| **练** | 产品力练习 | 基于探盘成果进行 AI 话术训练，快速掌握楼盘卖点 |
| **推** | 新媒体获客 | AI 生成小红书、公众号等平台推广内容，线上引流获客 |
| **约** | 现场邀约 | 录音上传分析，优化话术，提升邀约率和成交率 |

**产品 Slogan**：「探得真材，炼出专业，推向市场，约见成交」

### 1.3 核心价值

**顶层价值**
| 价值维度 | 具体描述 |
|----------|----------|
| **新媒体获客** | 通过探盘获取真实物料，AI 生成高质量内容，实现线上获客 |
| **产品力提升** | 基于楼盘的话术训练，帮助经纪人快速掌握产品卖点 |
| **管理可视化** | 实时追踪任务执行进度，数据看板辅助决策 |

**步骤价值（探-练-推-约）**
| 步骤 | 核心价值 | 说明 |
|------|----------|------|
| **探** | 真实物料，标准统一 | 标准化任务流程，AI自动整理物料 |
| **练** | AI训练，快速提升 | 自动关联题库，实时反馈，能力量化 |
| **推** | AI生成，降本增效 | 一键生成多平台文案 |
| **约** | 录音复盘，持续优化 | 量化话术能力，给出改进建议 |

### 1.4 项目范围

**PC管理后台用户**
| 用户角色 | 职责说明 |
|----------|----------|
| **新房项目维护人员** | 负责新房录入（需审核）、上线操作（无需审核） |
| **产品运营人员** | 负责AI话术训练题库准备、楼盘信息编辑 |
| **区域经理** | 负责任务下发、跟进追踪、数据分析；不能编辑楼盘信息 |

**小程序侧用户**
| 用户角色 | 职责说明 |
|----------|----------|
| **经纪人** | 执行探盘任务（收集物料、生成内容、话术训练、录音分析）、观测任务数据 |
| **区域经理** | 移动办公，负责任务下发、跟进追踪、数据分析 |

## 评审问题

### CEO 评审关注点
1. 产品定位是否符合公司战略方向？
2. 目标市场规模是否合理？
3. 竞争优势是否可持续？
4. 风险评估是否充分？

### PM 评审关注点
1. 用户痛点是否准确？
2. 产品定义是否清晰？
3. 功能范围是否合理？
4. 需求优先级是否明确？

### 产品 评审关注点
1. 产品价值主张是否清晰？
2. 用户旅程是否完整？
3. 竞品分析是否全面？
4. 产品差异化是否明显？
