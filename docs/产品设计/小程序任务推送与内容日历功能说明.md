## 一、产品概述

### 1.1 背景与目标

房产经纪人日常需要接收并执行多种任务：探盘打卡、话术训练、文案撰写、视频制作等。当前各任务分散在不同入口，缺乏统一的通知和执行入口。

**本期目标：** 在小程序侧建立统一的任务消息通知和执行流程，覆盖探盘打卡、话术训练、文案任务、视频任务、自定义任务、待下发任务六种任务类型，支持项目制任务管理。

### 1.2 核心场景

| 场景 | 描述 |
|------|------|
| 接收任务 | PC管理后台下发的任务自动推送到小程序 |
| 消息通知 | 日历页面顶部通知入口，聚合所有任务消息 |
| 执行任务 | 经纪人执行探盘打卡、话术训练、文案/视频制作等任务 |
| 进度追踪 | 下发人可跟进各阶段完成情况（后续规划） |

### 1.3 任务筛选设计

#### 1.3.1 身份切换（考虑项）

| 方案 | 说明 | 适用场景 |
|------|------|----------|
| 需要身份切换 | 显示"经纪人" / "管理员" 切换入口 | 管理功能与个人任务分离 |
| 不需要切换 | 始终展示所有待下发任务 | 简化流程，快速作业 |

**推荐**：暂不设计身份切换，后续按需扩展。

#### 1.3.2 时间维度

- **粒度**：按天展示
- **切换**：点击日历日期切换

#### 1.3.3 任务类型

| 类型 | 图标 | 说明 | 执行方式 |
|------|------|------|----------|
| 探盘任务 | 📍 | 到楼盘现场拍照、打卡、采集信息 | 4阶段分步引导 |
| 训练任务 | 📚 | AI话术训练 | 3阶段：阅读→开始→完成 |
| 文案任务 | ✏️ | 基于探盘素材生成文案 | 4阶段：待撰写→撰写中→已提交→已通过 |
| 视频任务 | 🎬 | 基于探盘素材制作视频 | 4阶段：待拍摄→剪辑中→待审核→已完成 |
| 自定义任务 | ⭐ | 用户给自己下发的任务 | 3阶段：进行中→已完成 |
| 待下发任务 | ⏳ | 管理员在移动端便捷下发 | 点击跳转任务下发页面 |

## 二、用户角色

| 角色 | 操作 |
|------|------|
| 经纪人 | 接收任务通知、执行探盘/训练/文案/视频/自定义任务 |
| 业务总监/区域经理 | 下发任务、追踪进度（后续规划） |
| 总部干部 | 配置任务模板、查看数据（后续规划） |

## 四、功能详述

### 4.1 日历页面通知入口

**位置：** calendar.html 顶部标题栏右侧

**样式：**
```
┌─────────────────────────────────────┐
│          任务中心           🔔(3)    │ ← 通知铃铛 + 数字红点
├─────────────────────────────────────┤
│  日历组件 (可折叠)                          │
├─────────────────────────────────────┤
│  [全部] [探盘] [训练] [文案] [视频] [自定义] [待下发] │ ← 7个Tab
├─────────────────────────────────────┤
│  今日任务 · 4月8日             3个任务 │
│  ┌─────────────────────────────────┐ │
│  │ [探盘卡片] [训练卡片] [文案卡片] │ │
│  └─────────────────────────────────┘ │
└─────────────────────────────────────┘
```

**交互说明：**

| 功能 | 说明 |
|------|------|
| 铃铛图标 | + 红色数字表示未读消息数量 |
| 点击铃铛 | → 展开通知面板弹窗 |
| 新任务到达 | → 自动弹出通知提示 |

### 4.2 通知面板

**形式：** 底部全宽弹窗，不跳转页面

**布局：**
```
┌─────────────────────────────────────┐
│  通知                [清空全部]  ×   │ ← 标题栏
├─────────────────────────────────────┤
│  [全部] [探盘] [训练] [文案] [视频] [自定义] [待下发] │ ← 7个Tab
├─────────────────────────────────────┤
│  ● 中海汇德里探盘任务                                    │
│    探盘任务待执行                          10分钟前    →
├─────────────────────────────────────┤
│  ○ 万科翡翠公园探盘任务                                   │
│    待下发任务已到达                        1小时前      →
├─────────────────────────────────────┤
│  ○ 议价谈判话术练习                                        │
│    训练任务待执行                          昨天        →
└─────────────────────────────────────┘
```

**功能说明：**

| 功能 | 说明 |
|------|------|
| 分类筛选 | 全部/探盘/训练/文案/视频/自定义/待下发 七个Tab |
| 时间排序 | 最新消息在前 |
| 已读/未读 | 圆点颜色区分（橙=未读，灰=已读） |
| 标记已读 | 点击消息后变为已读状态 |
| 清空全部 | 一键清除所有通知 |
| 关闭面板 | 点击×或遮罩层关闭 |

**消息卡片信息：**

| 字段 | 说明 |
|------|------|
| 任务图标 | ●/○ 圆点表示未读/已读 |
| 任务名称 | 任务标题 |
| 消息摘要 | 任务状态或提醒内容 |
| 发布时间 | 刚刚/10分钟前/昨天 等相对时间 |
| 跳转入口 | → 箭头，点击进入详情 |

### 4.4 探盘打卡执行流程

#### 4.4.1 四阶段说明

```
┌─────────────────────────────────────────────────────────────┐
│                    探盘任务四阶段流程                         │
└─────────────────────────────────────────────────────────────┘

  阶段1          阶段2          阶段3          阶段4
┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐
│  打卡  │ → │素材收集│ → │文案/脚本│ → │封面/视频│
└────────┘    └────────┘    └────────┘    └────────┘
    │              │              │              │
    ↓              ↓              ↓              ↓
 到达现场      拍照/视频      确认文案        封面+视频
 签到确认     /录音/文本      脚本内容        提交审核
                            (后台生成)     (后台生成)
```

**阶段详情：**

| 阶段 | 名称 | 必填 | 说明 |
|------|------|------|------|
| 阶段1 | 打卡 | ✅ | 到达楼盘现场，定位签到 |
| 阶段2 | 素材收集 | ✅ | 按配置的打卡点位完成资料采集（拍照/视频/录音/文本），必须完成才能提交 |
| 阶段3 | 文案/脚本生成 | - | 后台生成文案/脚本，经纪人确认 |
| 阶段4 | 封面图/视频生成 | - | 后台基于确认内容生成封面图+短视频 |

### 4.5 训练任务执行流程

训练任务独立于探盘任务，可直接开始训练。

**三阶段说明：**

| 阶段 | 名称 | 说明 |
|------|------|------|
| 1 | 阅读资料 | 阅读学习材料，了解话术要点 |
| 2 | 开始训练 | 开始AI话术训练 |
| 3 | 训练完成 | 完成训练，获得评分报告 |

**交互流程：**
```
训练卡片 → 点击"开始训练"/"继续训练"按钮 → speech-train.html
                                                    │
                                               选择项目
                                                    │
                                               选择模式
                                                    │
                                              开始训练
```

**卡片按钮状态：**

| 状态 | 按钮文字 | 说明 |
|------|----------|------|
| 待执行 | 开始训练 | 首次开始训练 |
| 进行中 | 继续训练 | 继续未完成的训练 |
| 已完成 | 查看报告 | 查看训练结果 |
