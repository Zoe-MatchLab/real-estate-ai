## 一、概述

### 1.1 页面定位

`checkin-point.html` 是**阶段2「点位打卡」**的核心页面，用户到达楼盘现场完成定位打卡（阶段1）后，进入此页面逐一完成各打卡点位的素材采集与上传。

### 1.2 核心功能

- 展示任务所有打卡点位及各自的审核状态
- 支持拍照、相册、视频、文字、语音、标签等多种素材类型上传
- AI 质量审核 + 人工审核流程
- 实时反馈审核结果（通过/失败原因+建议）
- 必须全部必填点位审核通过后，才可提交

### 1.3 与相邻页面的关系

```
task-detail.html (任务详情)
    └── [进入阶段2] ──→ checkin-point.html (本页面)
                              ├── [完成点位上传] ──→ confirm-script.html (文案确认)
                              └── [保存草稿] ──→ 返回 task-detail.html
```

## 三、点位卡片组件 · PointCard

### 3.1 状态分类

每个点位卡片有 **5 种审核状态**：

| 状态 | 标识 | 含义 |
|------|------|------|
| `pending` | ⏳ 无图标 | 待上传，从未上传过任何内容 |
| `uploading` | 🔄 进度环 | 上传中，有文件正在上传 |
| `reviewing` | 🔍 旋转图标 | AI 审核中，已上传正在审核 |
| `approved` | ✅ 绿色对勾 | 审核通过 |
| `rejected` | ⚠️ 橙色叹号 | 审核失败，显示原因+建议 |

### 3.2 卡片结构（展开态）

```
┌──────────────────────────────────────────────────────────────┐
│  [▶/▼] [必填/选填] [点位名称]  [类型图标]  [状态标识]  [操作] │
├──────────────────────────────────────────────────────────────┤
│  ┌──────────────────────────────────────────────────────────┐ │
│  │  内容预览区（已上传时显示）                               │ │
│  │  [缩略图/时长标签] [缩略图] [缩略图] ...                  │ │
│  └──────────────────────────────────────────────────────────┘ │
│                                                              │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │  操作按钮区                                               │ │
│  │  [📷 拍照] [🖼️ 相册] [🎬 视频]                          │ │
│  │  [📝 文字] [🎤 语音] [🏷️ 标签]                          │ │
│  └──────────────────────────────────────────────────────────┘ │
│                                                              │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │  审核状态区                                               │ │
│  │  [状态图标] [状态文字]                                   │ │
│  │  [失败原因 + 建议]（仅 rejected 状态）                    │ │
│  │  [AI建议区]（可选，显示 AI 补充信息）                     │ │
│  └──────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
```

### 3.3 卡片头部（收起态）

```
[▶] 必填：沙盘照片 (0/1)  📷  ⏳
```

- 左侧：展开/收起箭头
- 必填/选填标签：必填=橙色，选填=灰色
- 点位名称
- 类型图标：📷=拍照 🎬=视频 📍=区位 📝=文字 🎤=语音 🏷️=标签
- 已上传计数 (current/max)
- 状态标识 emoji

### 3.4 内容预览区

**图片/视频类型：**
- 网格布局，3列
- 每个缩略图 80×80px，圆角 8px
- 视频额外显示播放时长标签（如 `0:32`）
- 点击可预览/播放
- 长按/右上角菜单可删除

**文字类型：**
- 文字内容卡片，显示前 100 字
- 超出折叠，点击展开

**语音类型：**
- 音频波形图卡片
- 显示时长 + 播放按钮
- 右上角可删除

**标签类型：**
- 标签云展示
- 每个标签为胶囊形状

### 3.5 操作按钮区

每个点位支持的按钮由 PC 端配置决定（点位类型决定按钮组合）：

| 按钮 | 图标 | 触发行为 |
|------|------|----------|
| 拍照上传 | 📷 | 调用系统相机拍摄 |
| 相册选择 | 🖼️ | 打开相册选择已有图片 |
| 视频录制 | 🎬 | 调用系统相机录制视频 |
| 文字备注 | 📝 | 展开文字输入框 |
| 语音讲解 | 🎤 | 开始语音录制 |
| 添加标签 | 🏷️ | 弹出标签选择/新建面板 |

> 所有按钮均为 `button` 元素，带 `aria-label` 可访问性标签。

### 3.6 审核状态区

**pending（待上传）：**
```
状态文字：「请上传素材」
无额外提示
```

**uploading（上传中）：**
```
🔄 上传中...  ██████████░░░░░░░░  50%
```

**reviewing（AI审核中）：**
```
🔍 AI审核中...  ██████████████░░░  80%
```

**approved（审核通过）：**
```
✅ 审核通过
```

**rejected（审核失败）：**
```
⚠️ 审核未通过
原因：照片清晰度不足，无法识别楼盘信息
建议：请在光线充足的环境下重新拍摄，确保楼盘名称清晰可见
```

### 3.7 展开/收起交互

- 点击卡片头部区域切换展开/收起
- 展开时箭头旋转 90°，变为 `▼`
- 动画过渡：`height` + `opacity`，时长 250ms ease
- 一次只展开一个卡片（手风琴模式），或允许多个展开（待定，建议手风琴）

## 四、审核流程

### 4.1 完整流程图

```
用户上传素材
     ↓
  本地预览 + 立即显示 ⏳ pending
     ↓
 用户点击「完成」或「确认」
     ↓
 开始上传 ──→ 🔄 uploading（显示上传进度）
     ↓
 上传完成 ──→ 进入 AI 审核队列
     ↓
 🔍 AI审核中（reviewing）
     ↓
 ┌──────────────────────────────────┐
 │  AI 判断：                       │
 │  通过 ──────────────────→ ✅ approved
 │  不通过 ─────────────→ ⚠️ rejected
 │       ↓                              │
 │  显示失败原因 + 建议              │
 │  用户可重新上传                  │
 └──────────────────────────────────┘
```

### 4.2 AI 审核维度

| 维度 | 说明 | 失败原因示例 |
|------|------|------------|
| 清晰度 | 图片/视频是否清晰可辨 | 「照片模糊，请重新拍摄」 |
| 完整性 | 是否包含必要信息 | 「未拍摄到楼盘名称标识」 |
| 合规性 | 是否符合平台规范 | 「视频时长超过60秒限制」 |
| 水印 | 是否包含楼盘名称水印 | 「建议添加楼盘名称水印」 |

### 4.3 失败重传流程

```
⚠️ 审核失败
     ↓
 显示原因 + 建议
     ↓
 用户重新上传 / 修改
     ↓
 重新进入审核队列
     ↓
 再次 AI 审核
```

## 八、交互细节

### 8.1 文件上传限制（由 PC 端配置）

| 类型 | 限制 |
|------|------|
| 图片 | 最少 N 张，最多 M 张，单张 ≤ 10MB，支持 jpg/png/webp |
| 视频 | 最少 N 个，最多 M 个，时长 N~M 秒，≤ 100MB |
| 语音 | 最少 N 个，最多 M 个，时长 N~M 秒，≤ 20MB |
| 文字 | 字数限制 N~M 字 |
| 标签 | 最多 N 个 |

### 8.2 上传反馈

- 上传中：显示本地预览图 + 上传进度条
- 上传失败：缩略图显示红色角标 + 重试按钮
- 上传完成：进入 AI 审核队列

### 8.3 AI 审核轮询策略

- 上传完成后立即进入队列
- 前端每 3 秒轮询一次审核状态（WebSocket 更优）
- 审核完成后更新 UI + 推送通知

### 8.4 保存与离开

- 页面离开时自动保存草稿（`beforeunload` + `visibilitychange`）
- 保存按钮提供手动保存入口
- 未保存离开时给出确认提示

### 8.5 错误处理

| 场景 | 处理方式 |
|------|----------|
| 网络断开 | Toast 提示「网络已断开」，暂停上传，恢复后继续 |
| 上传超时 | 重试 3 次，失败后显示「上传失败，点击重试」|
| 文件格式不支持 | 立即 Toast 提示「不支持该文件格式」|
| 文件大小超限 | 立即 Toast 提示「文件超过大小限制」|

## 十、无障碍设计

| 要求 | 实现方式 |
|------|----------|
| 键盘导航 | 所有按钮/可交互元素可 Tab 聚焦 |
| 屏幕阅读器 | 按钮带 `aria-label`，状态变化时 `aria-live` 通知 |
| 色彩对比 | 文字与背景对比度 ≥ 4.5:1 |
| 焦点指示 | `:focus-visible` 高亮样式 |
| 动态内容 | `aria-live="polite"` 区域通知审核状态变化 |

## 十一、约束与假设

1. **不修改现有代码**：本文档仅输出设计规范，不修改 `task-detail.html` 等现有文件
2. **兼容性**：目标浏览器为微信小程序 webview，支持 ES6+
3. **数据来源**：点位配置、审核规则等由 PC 端下发，本页面负责展示与交互
4. **草稿保存**：草稿数据存储于 localStorage，正式提交时上传至服务器
5. **安全**：所有上传接口需校验用户身份和任务归属权
