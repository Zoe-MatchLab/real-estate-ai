## 一、概述与设计目标

### 1.1 页面定位

阶段4是用户在确认文案脚本后，查看**生成的封面图**和**宣传视频**成果的页面。用户可预览、下载、分享，最终提交审核。

### 1.2 设计目标

- **成果展示最大化**：封面图和视频占据页面主要视觉区域，够大够清晰
- **操作便捷**：下载和分享按钮直接放在对应成果下方，触手可及
- **完成感强**：顶部「4/4 已完成」标签 + 绿色成功色，给用户正向激励
- **移动端体验优先**：单列布局，竖屏视频播放器，封面图支持手势缩放

|------|------|------|------|

- 全屏预览遮罩 fade-in，图片 scale-in
- 下载中按钮显示 loading spinner
- 分享成功后显示成功 Toast

## 九、技术实现参考

- **视频**：`video` 标签 + `playsinline`（iOS WebKit 内联播放）
- **手势缩放**：使用 `Hammer.js` 或原生 `touch` 事件实现双指缩放
- **图片占位**：生成完成前显示 skeleton loader，避免页面跳动
- **安全区**：`env(safe-area-inset-bottom)` 适配 iPhone
- **下载**：使用 `<a download>` 或 `wx.downloadFile`（小程序）
- **分享**：Web 使用 Web Share API（`navigator.share`），小程序使用 `wx.showShareMenu`
- **字体缩放禁止**：`maximum-scale=1.0, user-scalable=no`
- **触摸反馈**：`active` 态加 `transform: scale(0.95~0.98)`
- **视频预加载**：封面图加载完成后开始预加载视频（不自动播放）
