## 一、产品定位

### 1.1 功能定位

让**区域经理/业务线总监**在小程序侧也能完成探盘任务下发，无需打开PC端，实现移动化办公。

### 1.2 核心场景

| 场景 | 描述 |
|------|------|
| 收到通知 | 新房项目上线后，区域经理收到推送 |
| 查看楼盘 | 点击通知查看楼盘信息 |
| 下发任务 | 选择楼盘后，配置执行人员并下发 |

### 1.3 与PC端差异

| 维度 | PC端 | 小程序端 |
|------|------|----------|
| 适用角色 | 运营人员、管理员 | 区域经理、业务线总监 |
| 操作复杂度 | 完整表单 | 简化下发流程 |
| 场景 | 日常管理 | 移动办公、应急下发 |

---

## 二、用户角色与权限

### 2.1 权限机制

PC端在上线新房时，通过**组织架构**选择可下发任务的人员（区域经理/业务总监）。

```
PC端操作：上线新房 → 选择可下发人员 → 发送通知
```

### 2.2 可下发人员

| 角色 | 操作 |
|------|------|
| 区域经理 | 收到通知 → 查看楼盘 → 下发任务 |
| 业务线总监 | 同上，可批量下发 |

**说明：** 只有被PC端配置为「可下发」的区域经理/业务总监才能在小程序侧操作

## 五、任务状态定义

### 5.1 任务状态说明

小程序下发任务后，任务进入与PC端统一的任务状态体系。

| 状态 | 说明 | 小程序端可见操作 |
|------|------|-----------------|
| 待分发 | 创建后未下发 | 下发、编辑、删除 |
| 待执行 | 已下发等待执行 | 查看详情、取消 |
| 进行中 | 经纪人在执行中 | 查看详情、催办 |
| 已提交 | 经纪人已提交，等待审核 | 查看详情、查看结果 |
| 已完成 | 审核通过 | 查看详情、查看结果 |
| 已驳回 | 审核驳回 | 查看详情、重新下发 |
| 已逾期 | 超时未完成 | 查看详情、继续执行 |

### 5.2 状态流转

```
待分发 → 待执行 → 进行中 → 已提交 → 已完成/已驳回
                          ↘ 已逾期（可继续执行）
```

### 5.3 审核与逾期说明

**审核时机：**
- 执行人员在微信小程序完成所有阶段执行后，点击「提交」，任务状态变为「已提交」
- 只有「已提交」状态的任务，审核人员才能进行审核操作
- 审核通过 → 状态变为「已完成」
- 审核驳回 → 状态变为「已驳回」，执行人员修改后可重新提交

**逾期说明：**
- 超过截止时间未完成的任务，状态自动变为「已逾期」
- 已逾期任务仍可继续执行，执行后状态恢复为「进行中」

## 七、与PC端下发关系

### 7.1 数据共用

**小程序下发的任务和PC下发的任务共用数据**，同一张任务表，可在任意一端查看和操作。

| 维度 | PC端 | 小程序端 |
|------|------|----------|
| 入口 | PC任务列表 → 新建 | 小程序通知 → 楼盘详情 |
| 表单 | 完整表单（可多楼盘） | 简化表单（单楼盘） |
| 楼盘选择 | 下拉选择已有楼盘 | 直接展示上线楼盘 |
| 执行人员 | 下拉多选 | 勾选列表（组织架构） |
| 适用场景 | 日常管理、批量操作 | 移动办公、应急下发 |
| 审核流程 | 无需审核，直接下发 | 无需审核，直接下发 |

### 7.2 任务来源标识

任务列表中通过标签区分来源：

| 来源 | 标签样式 |
|------|----------|
| PC下发 | 灰色标签「PC」 |
| 小程序下发 | 橙色标签「小程序」 |

### 7.3 审核说明

**探盘任务下发无需审核**，配置完成后直接下发到执行人员。

## 九、数据结构

### 9.1 新房上线通知

```javascript
{
  notifyId: "NOTIFY_xxx",
  type: "house_online",  // 新房上线
  houseId: "HOUSE_xxx",
  houseName: "中海汇德里",
  developer: "中海地产",
  message: "新房项目已上线，请配置探盘任务",
  isRead: false,
  // 可下发人员（PC端配置）
  allowDistributeUsers: ["USER_001", "USER_002"],
  createTime: "2026-03-31T21:00:00Z"
}
```

### 9.2 探盘任务

```javascript
{
  taskId: "TASK_xxx",
  // 数据与PC共用
  source: "miniapp",  // 来源：pc/miniapp
  
  taskName: "中海汇德里新盘探盘",
  houseId: "HOUSE_xxx",
  
  // 楼盘信息（不可编辑，仅展示）
  house: {
    name: "中海汇德里",
    address: "上海市浦东新区张江路",
    price: "8.5万/㎡"
  },
  
  priority: "medium",
  deadline: "2026-04-07T18:00:00Z",
  
  // PC端打卡模版配置
  checkInConfig: {
    radius: 500,  // 打卡范围半径（米）
    rule: "location_first",  // 先定位打卡→再点位打卡
    completionRule: "all_required",  // all_required / half_required
    timeLimit: 7200,  // 点位打卡完成时间限制（秒）
    resultFlow: ["copywriting", "video_script", "cover_image", "video"]  // 结果流向
  },
  
  // PC端打卡点位配置
  checkPoints: [
    { id: 1, name: "沙盘打卡点", required: true, contentTypes: ["photo", "video"] },
    { id: 2, name: "样板间打卡点", required: true, contentTypes: ["photo", "video", "audio"] },
    { id: 3, name: "区位配套打卡点", required: false, contentTypes: ["photo", "text"] }
  ],
  
  executorIds: ["USER_001", "USER_002", "USER_003"],
  
  // 状态
  status: "pending_execute",  // 待分发/待执行/进行中/已提交/已完成/已驳回/已逾期
  createTime: "2026-03-31T21:00:00Z",
  distributeTime: "2026-03-31T21:05:00Z"
}
```
