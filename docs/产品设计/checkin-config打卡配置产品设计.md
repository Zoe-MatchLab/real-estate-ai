## 一、产品概述

### 1.1 页面定位

打卡配置页面用于管理员/经纪人下发探盘任务时，配置任务是否需要打卡以及打卡点位。探盘人员到达楼盘后需要按配置的打卡点进行签到打卡。

### 1.2 核心功能

| 模块 | 说明 | 状态 |
|------|------|------|
| 任务基本信息 | 显示任务名称、类型、时间 | ✅ 已实现 |
| 打卡开关 | 配置是否需要打卡 | ✅ 已实现 |
| 打卡点位列表 | 显示所有打卡点 | ✅ 已实现 |
| 打卡点增删改 | 添加/删除/修改打卡点 | ✅ 已实现 |
| 点位元素管理 | 配置打卡点的具体元素 | ✅ 已实现 |
| 保存/发布 | 保存配置并下发任务 | ✅ 已实现 |

#### 3.4.1 区域布局

```
┌─────────────────────────────────────────┐
│  📍 打卡点位配置               [+ 添加]  │

│  │  必填：定位                       │   │
│  │  选填：拍照  |  签字  |  备注     │   │

│                                         │
│  ┌─────────────────────────────────┐   │
│  │ 🏠  样板间入口         [编辑][删除]│ │
│  │  必填：定位  |  拍照             │   │
│  │  选填：签字  |  备注             │   │
│  └─────────────────────────────────┘   │
│                                         │
└─────────────────────────────────────────┘
```

#### 3.4.2 打卡点数据结构

```javascript
{
  id: "point_001",           // 打卡点唯一ID
  name: "售楼处门口",         // 打卡点名称
  location: {                 // 定位信息
    latitude: 30.5721,       // 纬度
    longitude: 114.1234,     // 经度
    address: "武汉市洪山区..." // 地址
  },
  elements: [                 // 点位元素
    { type: "location", required: true, name: "定位" },
    { type: "photo", required: true, name: "拍照" },
    { type: "sign", required: false, name: "签字" },
    { type: "remark", required: false, name: "备注" }
  ],
  remark: ""                 // 打卡点备注
}
```

#### 3.4.3 打卡点位元素类型

| 元素类型 | 标识 | 必填 | 说明 |
|----------|------|------|------|
| 定位 | location | 是 | 必须获取GPS定位 |
| 拍照 | photo | 否 | 必须拍照上传 |
| 签字 | sign | 否 | 需要手写签名 |
| 备注 | remark | 否 | 可填写文字说明 |

**元素配置规则**：
- 至少配置1个必填元素
- 定位元素默认为必填，不可取消

## 六、交互逻辑

### 6.1 初始化流程

```
1. 页面加载
   │
   ├── 读取URL参数获取taskId
   │
   ├── 调用接口获取任务详情
   │
   ├── 调用接口获取已有打卡配置（如有）
   │
   └── 渲染页面
```

### 6.2 保存流程

```
1. 点击「保存」或「保存并下发」
   │
   ├── 校验必填项
   │     ├── 打卡开关开启时，必须至少1个打卡点
   │     └── 每个打卡点必须有名称和定位
   │
   ├── 校验失败 → 提示错误信息
   │
   ├── 校验成功
   │     ├── 收集页面数据
   │     ├── 调用保存接口
   │     │
   │     └── 保存成功
   │           ├── 提示"保存成功"
   │           └── 返回上一页
```

### 6.3 删除打卡点流程

```
1. 点击「删除」按钮
   │
   ├── 弹出确认框
   │
   ├── 用户取消 → 关闭弹窗
   │
   ├── 用户确认删除
   │     ├── 从列表移除
   │     ├── 重新编号
   │     └── 更新页面
```
