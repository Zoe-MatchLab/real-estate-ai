## 一、产品概述

### 1.1 页面定位

打卡配置页面用于管理员下发探盘任务时，配置任务的执行人员、通知人员、截止时间以及打卡点位。执行人员到达楼盘后需要按配置的打卡点进行签到打卡。

### 1.2 核心功能

| 模块 | 说明 | 状态 |
|------|------|------|
| 打卡配置 | 定位打卡开关、打卡范围 | ✅ 已实现 |
| 执行人员 | 组织机构树选择执行人 | ✅ 已实现 |
| 通知人员 | 组织机构树选择需通知的人 | ✅ 已实现 |
| 截止时间 | 设置任务完成截止时间 | ✅ 已实现 |
| 打卡点位 | 支持标题/描述/类型/必填/最少数量 | ✅ 已实现 |
| 保存下发 | 保存配置并触发任务下发 | ✅ 已实现 |

## 三、功能详解

### 3.1 打卡配置

**布局**：
```
┌─────────────────────────────────────────┐
│  定位打卡                            [●]│
│  开启后，需到达指定位置才能打卡           │
│                                         │
│  允许偏差范围 [ 100 ] 米                │
└─────────────────────────────────────────┘
```

**交互**：

| 元素 | 交互 | 效果 |
|------|------|------|
| 开关 | 点击切换 | 开启/关闭定位打卡功能 |
| 范围输入 | 输入数字 | 设置允许的GPS偏差范围（10-1000米） |

**样式**：

| 属性 | 值 |
|------|---|
| 开关关闭态 | 灰色背景 #E5E6EB |
| 开关开启态 | 橙色背景 #FA8C16 |
| 范围输入 | 数字输入框，默认值100米 |

### 3.5 打卡点位配置

#### 3.5.1 打卡点数据结构

```javascript
{
  id: "point_001",           // 打卡点唯一ID
  name: "沙盘照片",           // 打卡点标题
  desc: "拍摄沙盘全景照片...", // 描述说明
  types: ["photo", "tag"],   // 上传文件类型
  required: true,            // 是否必填
  minCount: 1               // 最少完成数量
}
```

#### 3.5.2 上传类型

| 类型 | 标识 | 图标 | 说明 |
|------|------|------|------|
| 拍照 | photo | 📷 | 拍摄现场照片 |
| 视频 | video | 🎬 | 录制视频 |
| 文字 | text | 📝 | 填写文字说明 |
| 语音 | audio | 🎤 | 录制语音 |
| 标签 | tag | 🏷️ | 选择预设标签 |

**说明**：
- 至少选择1种类型
- 「拍照」默认为必选（打卡必须拍照留证）

#### 3.5.3 添加/编辑弹窗

```

│  添加打卡点                        [×]   │

│  ┌─────────────────────────────────┐   │
│  │ 如：沙盘照片、样板间视频        │   │
│  └─────────────────────────────────┘   │
│                                         │
│  描述说明                               │
│  ┌─────────────────────────────────┐   │

│  └─────────────────────────────────┘   │
│                                         │

│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐│
│  │📷拍照│ │🎬视频│ │📝文字│ │🎤语音│ │🏷️标签││
│  └─────┘ └─────┘ └─────┘ └─────┘ └─────┘│
│                                         │
│  ☑ 设为必填项                           │
│                                         │
│  最少完成数量                           │
│  ┌───────┐ 个点位                       │
│  │   1   │                             │
│  └───────┘                             │
│                                         │
├─────────────────────────────────────────┤
│        [ 确认添加 ]                     │
└─────────────────────────────────────────┘
```

**字段说明**：

| 字段 | 必填 | 说明 |
|------|------|------|
| 打卡点标题 | ✅ 是 | 1-20个字符，如"沙盘照片" |
| 描述说明 | 否 | 最多200字符，对执行人员的指导说明 |
| 上传类型 | ✅ 是 | 至少选择1种，默认拍照 |
| 设为必填 | 否 | 必填项需完成才能提交任务 |
| 最少完成数量 | 是 | 默认1，限制执行端最少完成数 |

#### 3.5.4 checkin-point.html 的数据来源

打卡点配置将传递给 `checkin-point.html`（执行端页面）：

```javascript
// checkin-config.html 保存的配置
{
  points: [
    {
      id: 1,
      name: '沙盘照片',
      desc: '拍摄沙盘全景照片，清晰展示项目名称',
      types: ['photo'],
      required: true,
      minCount: 1
    },
    {
      id: 2,
      name: '区位配套',
      desc: '拍摄周边交通、商业配套',
      types: ['photo', 'text'],
      required: false,
      minCount: 1
    }
  ]
}

// checkin-point.html 接收并渲染
```

**checkin-point.html 的校验逻辑**：
1. 检查必填项是否全部完成
2. 检查每种类型的完成数量是否 >= minCount
3. 只有全部校验通过才能提交

## 六、校验规则

### 6.1 必填校验

| 校验项 | 条件 | 提示 |
|--------|------|------|
| 执行人员 | 至少选择1人 | 请选择执行人员 |
| 打卡点 | 至少1个 | 请添加至少一个打卡点 |
| 打卡点标题 | 不能为空 | 请输入打卡点标题 |
| 上传类型 | 至少1种 | 请至少选择一个上传类型 |

### 6.2 checkin-point.html 校验

```javascript
function validateSubmission() {
  var requiredPoints = config.points.filter(p => p.required);
  var optionalPoints = config.points.filter(p => !p.required);
  
  // 1. 必填项全部完成
  for (var p of requiredPoints) {
    if (getCompletedCount(p) < p.minCount) {
      return { valid: false, message: '请完成所有必填项' };
    }
  }
  
  // 2. 总完成数 >= 最少要求
  var totalCompleted = config.points.reduce((sum, p) => {
    return sum + Math.min(getCompletedCount(p), p.minCount);
  }, 0);
  
  var totalMinCount = config.points.reduce((sum, p) => sum + p.minCount, 0);
  
  if (totalCompleted < totalMinCount) {
    return { valid: false, message: '还需完成 ' + (totalMinCount - totalCompleted) + ' 个点位' };
  }
  
  return { valid: true };
}
```
