## 二、流程详解

### 2.1 第一阶段：新盘管理

**负责角色**：新房维护人员、审核人员

**流程**：

```
新房资料入库 → 提交审核 → 审核通过 → 新盘上线 → 通知可下发探盘任务的人员
```

**参与模块**：

- 新盘管理（PC）

**说明**：

1. 新房维护人员将新盘资料录入系统
2. 提交审核后，审核人员审核通过
3. 审核通过后新盘自动上线
4. 系统通知可下发探盘任务的运营人员/区域经理

---

### 2.2 第二阶段：探盘任务

**负责角色**：运营人员、经纪人（执行人）、审核人员

**流程**：

```
配置任务 ──→ 下发 ──→ 执行（阶段1定位打卡） ──→ 执行（阶段2点位打卡）
    │
    ├──→ 执行（阶段3文案确认） ──→ 执行（阶段4成果生成） ──→ 提交 ──→ 审核
    │                                                    │
    │                                                    ↓
    │                                            探盘资料自动同步
    │                                            到楼盘物料库
    │
    └──────────────────────────────────────────────────┘
                                         │
                                         ↓
                              执行人收到审核结果通知
```

**参与模块**：

- PC探盘任务下发（PC）
- 小程序探盘任务下发（小程序）
- 小程序探盘任务执行（小程序）
- 小程序任务消息通知与执行流程（小程序）

**⚠️ 后台执行说明**：
```
阶段4a: 生成文案脚本
    ↓
用户点击「生成」→ 开始后台执行
    ↓
┌─────────────────────────────────────┐
│  ✅ 用户可返回任务列表              │
│  ✅ 可操作其他小程序页面            │
│  ✅ 可退出小程序                    │
│  ✅ 后台服务持续运行                │
│  ✅ 任务卡片显示「生成中...」        │
└─────────────────────────────────────┘
    ↓
生成完成 → 推送通知 → 用户确认
    ↓
阶段4b: 生成封面视频（同理后台执行）
```

**四阶段执行说明**：

| 阶段 | 名称 | 执行端 | 说明 |
|------|------|--------|------|
| 1 | 定位打卡 | 小程序 | 到达楼盘现场，定位签到确认 |
| 2 | 点位打卡 | 小程序 | 按配置完成各点位资料采集（拍照/视频/录音/文本） |
| 3 | 文案确认 | 小程序 | 确认AI生成的文案脚本 |
| 4a | 生成文案脚本 | **小程序端（后台执行）** | 按配置完成各点位资料采集后，在小程序端生成多平台文案和短视频脚本 |
| 4b | 生成封面视频 | **小程序端（后台执行）** | 用户**确认文案脚本后**，再生成封面图和视频 |
| 提交 | 提交审核 | 小程序 | 确认成果后提交PC端审核 |

**⚠️ 后台执行说明（重要）**：
- 阶段4a/4b 生成过程中：
  - ✅ **用户可返回任务列表（内容日历页）**
  - ✅ 用户可操作小程序其他页面
  - ✅ 用户可退出小程序
  - ✅ **后台接口服务持续运行**，不受前端页面切换影响
- **任务卡片实时显示阶段状态**（如"生成中 30%"、"文案生成完成"）
- 生成完成后**自动推送通知**给用户

**审核结果**：

- 通过：状态变为「已完成」，探盘资料同步到楼盘物料库
- 驳回：状态变为「已驳回」，执行人修改后重新提交

---

### 2.3 第三阶段：话术训练

**负责角色**：经纪人（训练人）

**前置条件**：新房上线时，系统自动关联该楼盘的训练项目

**流程**：

```
收到通知 ──→ 点击训练入口 ──→ AI训练对话 ──→ 完成训练
```

**参与模块**：

- 小程序话术训练（小程序执行）

**说明**：

1. 新房上线时，系统自动关联该楼盘对应的训练项目（预配置）
2. 探盘任务下发后，经纪人收到通知
3. 经纪人点击任务卡片上的「产品力训练」入口
4. 进入AI话术训练，选择场景进行对话练习
5. 训练完成后记录完成状态

**话术训练与探盘任务的关系**：

- 探盘任务下发 → 经纪人收到通知，同时收到训练入口提醒
- 话术训练帮助经纪人熟悉楼盘 → 更好地完成探盘任务
- 探盘任务完成 → 积累楼盘知识 → 话术训练进一步巩固

## 四、任务类型总览

| 任务类型 | 配置入口 | 下发对象 | 说明 |
|----------|----------|----------|------|
| 探盘任务 | 新盘管理 → 探盘任务 | 经纪人 | 新盘上线后的线下探盘采集任务 |

## 五、状态体系统一

### 5.1 任务状态定义（按模块区分）

#### 探盘任务状态（7种）

| 状态 | 说明 | 适用 |
|------|------|------|
| 待分发 | 任务创建后未下发 | 探盘任务 |
| 待执行 | 已下发等待执行 | 探盘任务 |
| 进行中 | 执行中，有阶段未完成 | 探盘任务 |
| 已提交 | 已提交等待审核 | 探盘任务 |
| 已完成 | 审核通过 | 探盘任务 |
| 已驳回 | 审核驳回需修改 | 探盘任务 |
| 已逾期 | 超时未完成 | 探盘任务 |

#### 话术训练状态（2种）

| 状态 | 说明 | 适用 |
|------|------|------|
| 待训练 | 训练项目已关联，尚未开始练习 | 话术训练 |
| 已完成 | 已完成至少一次练习 | 话术训练 |

**说明**：话术训练是自主练习功能，不涉及下发、审核、驳回、逾期等概念

### 5.2 状态流转统一图

```
┌─────────────────────────────────────────────────────────────┐
│                    统一任务状态流转                            │
└─────────────────────────────────────────────────────────────┘

  待分发（仅探盘/话术）
      │
      ↓
  待执行 ──→ 进行中 ──→ 已提交 ──→ 已完成
                  │           │
                  │           ↓
                  │        已驳回 ──→ 进行中
                  │
                  └───────────→ 已逾期
```

## 六、通知体系

| 通知类型 | 触发时机 | 接收对象 | 说明 |

- 日历页面 → 通知铃铛 → 通知面板（聚合所有通知）

## 七、关键业务规则

### 7.1 探盘任务前置条件

- 探盘任务基于已上线的新盘创建
- 新盘上线后，系统通知可下发人员配置探盘任务

### 7.2 话术训练与探盘的关系

- 探盘任务下发后，经纪人可基于探盘成果生成专属话术
- 专属话术用于约看时的专业讲解

### 7.3 内容发布与探盘的关系

- 内容发布依赖探盘任务的成果
- 探盘任务未完成，内容发布任务锁定

### 7.4 约看与探盘/话术的关系

- 探盘任务完成 → 经纪人对楼盘足够了解 → 可进行约看
- 话术训练通过 → 掌握专业话术 → 提升约看成功率
