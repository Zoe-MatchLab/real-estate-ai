## 一、产品定位

### 1.1 功能定位

PC端探盘任务下发是**探盘任务管理模块**的核心功能，运营人员在PC端创建/配置探盘任务，指定执行人员和通知人员后下发，经纪人在小程序侧接收任务并执行。

### 1.2 核心价值

| 价值 | 说明 |
|------|------|
| 标准化下发 | 运营人员批量选择楼盘、配置人员后一键下发 |
| 责任到人 | 明确执行人员，任务可追溯 |
| 实时同步 | 下发后小程序侧即时收到通知 |
| 阶段管控 | 4阶段执行流程，进度可视化 |

---

## 二、用户角色

| 角色 | 操作 |
|------|------|
| 运营人员 | 创建任务、选择楼盘、配置人员、下发 |
| 业务总监 | 查看任务进度、催办执行 |
| 审核人员 | 审核探盘结果 |
| 管理员 | 全局管理、配置 |

---

### 4.2 新建/编辑任务表单

**触发：** 点击「新建探盘任务」按钮

**表单字段：**

| 模块 | 字段 | 说明 | 必填 |
|------|------|------|------|
| **基础信息** | 任务名称 | 如"中海汇德里新盘探盘" | ✅ |
| | 关联楼盘 | 从已上线楼盘中选择，可多选 | ✅ |
| | 优先级 | 高/中/低 | ✅ |
| | 截止时间 | 日期时间选择器 | ✅ |
| | 探盘要求 | 填写具体要求、收集的信息 | ✅ |
| **人员配置** | 执行人员 | 负责线下探盘的经纪人，最多10人 | ✅ |
| | 通知人员 | 知晓任务进度的知会人员 | ❌ |
| **点位配置** | 打卡要求 | 是否需要定位打卡 | ❌ |
| | 必填点位 | 拍照/视频/录音/文本，可多选 | ✅ |
| | 选填点位 | 同上，可多选 | ❌ |

**交互特性：**
- 关联楼盘后自动带入楼盘基础信息
- 执行人员选择后自动关联通知人员（可选）
- 必填点位至少选择一项
- 支持草稿自动保存

---

### 4.3 任务下发

**触发：** 列表页点击「下发」按钮，或勾选多个任务后「批量下发」

**下发流程：**
```
┌─────────────────────────────────────────┐
│           任务下发                        │
├─────────────────────────────────────────┤
│                                         │
│  选择执行人员：                           │
│  ┌─────────────────────────────────┐   │
│  │ [头像] 张三  经纪人              │   │
│  │ [头像] 李四  经纪人              │   │
│  │ [头像] 王五  经纪人              │   │
│  └─────────────────────────────────┘   │
│                                         │
│  已选：3人                              │
│                                         │
│  [取消]              [确认下发]          │
└─────────────────────────────────────────┘
```

**下发后：**
- 执行人员收到小程序通知
- 任务状态变为「待执行」
- 通知人员收到知会通知

---

### 4.4 任务状态

| 状态 | 说明 | PC端可见操作 |
|------|------|-------------|
| 待分发 | 创建后未下发 | 下发、编辑、删除 |
| 待执行 | 已下发等待执行 | 查看详情、催办、取消 |
| 执行中 | 经纪人在执行中 | 查看进度、催办 |
| 已提交 | 经纪人已提交 | 查看结果、审核 |
| 已完成 | 审核通过 | 查看结果 |
| 已驳回 | 审核驳回 | 重新下发 |

## 五、与小程序侧联动

### 5.1 联动流程

```
PC端                          小程序侧
──────────────────────────────────────────────────
创建任务
   ↓
选择楼盘 ──────────────────────────────────────
   ↓
配置人员 ──────────────────────────────────────
   ↓
点击「下发」 ──────→ 收到任务通知
   ↓                      ↓
状态变为「待执行」     进入任务详情
   ↓                      ↓
                   执行阶段1：定位打卡
   ↓                      ↓
                   执行阶段2：采集执行
   ↓                      ↓
                   执行阶段3：文案确认（后台生成）
   ↓                      ↓
                   执行阶段4：成果生成（后台生成）
   ↓                      ↓
提交审核 ←──────────────── 提交结果
   ↓
审核通过
   ↓                      ↓
状态变为「已完成」     收到审核通过通知
```

### 5.2 通知机制

| 触发时机 | 通知对象 | 通知内容 |
|----------|----------|----------|
| 任务下发 | 执行人员 | 您有新的探盘任务 |
| 任务下发 | 通知人员 | 探盘任务已下发 |
| 截止提醒 | 执行人员 | 任务即将到期 |
| 超时催办 | 执行人员 | 任务已超时，请尽快完成 |
| 审核通过 | 执行/通知人员 | 探盘任务已完成 |
| 审核驳回 | 执行人员 | 探盘结果被驳回 |

## 七、数据结构

### 7.1 探盘任务

```javascript
{
  taskId: "TASK_xxx",
  taskName: "中海汇德里新盘探盘",
  
  // 关联信息
  houses: [
    { houseId: "H001", houseName: "中海汇德里" }
  ],
  
  // 优先级
  priority: "high",  // high/medium/low
  
  // 状态
  status: "pending_distribute",  // 待分发/待执行/执行中/已提交/已完成/已驳回
  
  // 人员
  executors: [
    { id: "USER_001", name: "张三", stage: 2, progress: 50 }
  ],
  notifiers: [
    { id: "USER_002", name: "小明" }
  ],
  
  // 配置
  requireCheckIn: true,
  requiredPoints: ["photo", "video"],
  optionalPoints: ["audio", "text"],
  
  // 时间
  createTime: "2026-03-31T08:00:00Z",
  distributeTime: "2026-03-31T09:00:00Z",
  deadline: "2026-03-31T18:00:00Z",
  
  // 执行进度
  progress: {
    completed: 1,  // 已完成人数
    total: 3       // 总人数
  }
}
```

### 7.2 探盘结果

```javascript
{
  resultId: "RESULT_xxx",
  taskId: "TASK_xxx",
  executorId: "USER_001",
  
  // 各阶段状态
  stages: {
    stage1: { status: "completed", checkInTime: "..." },
    stage2: { status: "completed", points: [...] },
    stage3: { status: "completed", confirmed: true },
    stage4: { status: "completed", coverUrl: "...", videoUrl: "..." }
  },
  
  submitTime: "2026-03-31T15:00:00Z",
  
  // 审核
  audit: {
    status: "pending",  // pending/approved/rejected
    auditorId: null,
    opinion: null,
    auditTime: null
  }
}
```
