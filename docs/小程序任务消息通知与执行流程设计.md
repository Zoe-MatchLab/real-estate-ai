# 小程序任务消息通知与执行流程设计

> 创建时间：2026-03-31  
> 状态：待评审

---

## 1. 功能概述

### 1.1 产品定位

实现小程序侧任务消息通知和执行流程，覆盖探盘打卡、话术训练、内容发布三种任务类型，支持项目制任务（多个子任务归属一个探盘任务）。

### 1.2 核心价值

- **统一消息入口**：首页顶部通知铃铛，聚合所有任务消息
- **项目制任务**：一个探盘任务包含多个子任务，子任务可独立执行
- **阶段进度追踪**：可视化进度条，下发人可跟进各阶段完成情况

## 2. 页面结构

### 2.1 页面清单

| 页面 | 文件 | 说明 |
|------|------|------|
| AI工具（首页） | tools.html | 顶部增加通知入口 |

## 3. 通知入口设计

### 3.1 首页通知入口（tools.html）

```
┌─────────────────────────────────────┐
│ 🔔 通知(3)              🔍  │ ← 顶部工具栏
├─────────────────────────────────────┤
│                                     │
│         AI工具网格                  │
│                                     │
└─────────────────────────────────────┘
```

**交互：**
- 点击通知铃铛 → 展开通知面板
- 通知数字红点显示未读消息数量
- 有新任务时自动弹出通知提示

### 3.2 通知面板布局

```
┌─────────────────────────────────────┐
│ 📋 通知                    [清空] × │
├─────────────────────────────────────┤
│ [全部] [探盘] [话术] [内容]         │ ← 分类Tab
├─────────────────────────────────────┤
│ 🔴 中海汇德里探盘任务                 │
│    2/4阶段进行中，请注意              │
│    刚刚                          → │
├─────────────────────────────────────┤
│ ⚪ 万科城市之光话术训练                │
│    新任务已下发                      │
│    10分钟前                     → │
├─────────────────────────────────────┤
│ ⚪ 朋友圈文案生成                     │
│    等待探盘完成后执行                  │
│    昨天                         → │
└─────────────────────────────────────┘
```

**功能说明：**

| 功能 | 说明 |
|------|------|
| 分类筛选 | 全部/探盘/话术/内容 四个Tab切换 |
| 时间排序 | 最新消息在前 |
| 标记已读 | 点击消息后变为灰色（已读） |
| 清空功能 | 一键清空全部通知 |
| 跳转详情 | 点击右侧箭头或卡片进入任务详情 |

## 4. 探盘任务卡片设计

### 4.1 卡片布局

```
┌─────────────────────────────────────────┐
│  🏠 中海汇德里探盘任务           [进行中] │
├─────────────────────────────────────────┤
│                                         │
│  📸 探盘打卡           [待执行]    →    │
│  💬 话术训练           [已完成]    →    │
│  📝 内容发布           [待解锁]    →    │
│                                         │
│  进度：2/4 阶段                          │
│  ████████░░░░░░░░░░░░░░░░░░░░  50%   │
│                                         │
└─────────────────────────────────────────┘
```

### 4.2 子任务入口规则

| 子任务 | 入口 | 状态说明 |
|--------|------|----------|
| 📸 探盘打卡 | task-detail.html | 待执行/进行中/已完成 |
| 💬 话术训练 | speech-train.html | 可直接开始（后台配置时绑定） |
| 📝 内容发布 | 提示"内容发布需先完成探盘打卡" | 待解锁/待执行/已完成 |

**依赖关系：**
- 话术训练：可直接开始，独立于探盘打卡
- 内容发布：必须探盘打卡完成才能解锁执行

## 5. 探盘打卡执行流程（task-detail.html）

### 5.1 四阶段流程

```
┌─────────────────────────────────────────────────────────────┐
│                    探盘任务执行流程                          │
└─────────────────────────────────────────────────────────────┘

  阶段1        阶段2        阶段3        阶段4
┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐
│定位打卡│ → │采集执行│ → │文案确认│ → │成果生成│
└──────┘    └──────┘    └──────┘    └──────┘
  必填        拍照/视频    确认脚本    封面+视频
             /录音/文本   文案内容    提交审核
```

### 5.2 各阶段说明

| 阶段 | 内容 | 说明 |
|------|------|------|
| 阶段1：定位打卡 | 到达楼盘现场，签到打卡 | 定位确认当前位置 |
| 阶段2：采集执行 | 拍照/视频/录音/文本 | 后台可配置必填点位，不完成不能提交 |
| 阶段3：文案确认 | 确认后台生成的文案/脚本 | 后台生成后显示在任务卡片 |
| 阶段4：成果生成 | 封面图 + 短视频 | 基于确认的文案内容自动生成 |

## 6. 话术训练跳转

### 6.1 跳转规则

话术训练任务直接跳转 `speech-train.html`，无需经过探盘打卡。

### 6.2 交互流程

```
通知列表 → 点击话术任务 → speech-train.html
                              ↓
                         选择项目
                              ↓
                         选择模式
                              ↓
                        开始训练
```

---

## 7. 内容发布提示

### 7.1 解锁条件

内容发布必须探盘打卡完成（阶段2）才能解锁。

### 7.2 各状态提示

| 状态 | 提示文案 |
|------|----------|
| 待解锁 | "内容发布需先完成探盘打卡" |
| 待执行 | "请确认文案脚本后生成内容" |
| 进行中 | "正在生成封面和短视频..." |
| 已完成 | "内容已提交审核" |

  taskId: "TASK_xxx",
  taskName: "中海汇德里探盘任务",
  taskType: "exploration", // exploration/speech/content
  status: "progress", // pending/progress/completed/timeout
  stages: {
    current: 2,        // 当前阶段 1-4

## 10. 页面交互汇总

| 入口 | 行为 |
|------|------|
| 首页通知铃铛 | 展开通知面板弹窗 |
| 通知列表项 | 标记已读 + 跳转任务卡片 |
| 探盘任务卡片 | 进入task-detail.html |
| 话术训练入口 | 跳转speech-train.html |

| 内容发布（未解锁） | 提示"需先完成探盘打卡" |
