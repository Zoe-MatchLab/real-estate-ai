## 一、产品定位

### 1.1 功能定位

让**区域经理/业务线总监**在小程序侧也能完成探盘任务下发，无需打开PC端，实现移动化办公。

### 1.2 核心场景

| 场景 | 描述 |
|------|------|
| 收到通知 | 新房项目上线后，区域经理收到推送 |
| 查看楼盘 | 点击通知查看楼盘信息 |
| 下发任务 | 选择楼盘后，配置执行人员并下发 |

### 1.3 与PC端差异

| 维度 | PC端 | 小程序端 |
|------|------|----------|
| 适用角色 | 运营人员、管理员 | 区域经理、业务线总监 |
| 操作复杂度 | 完整表单 | 简化下发流程 |
| 场景 | 日常管理 | 移动办公、应急下发 |

---

## 二、用户角色与权限

### 2.1 权限机制

PC端在上线新房时，通过**组织架构**选择可下发任务的人员（区域经理/业务总监）。

```
PC端操作：上线新房 → 选择可下发人员 → 发送通知
```

### 2.2 可下发人员

| 角色 | 操作 |
|------|------|
| 区域经理 | 收到通知 → 查看楼盘 → 下发任务 |
| 业务线总监 | 同上，可批量下发 |

**说明：** 只有被PC端配置为「可下发」的区域经理/业务总监才能在小程序侧操作

### 4.3 配置探盘任务

**触发：** 点击「配置探盘任务并下发」

**说明：** 小程序配置内容**与PC端保持一致**

**表单字段说明：**

| 字段 | 说明 | 必填 |
|------|------|------|
| 任务名称 | 自动生成「楼盘名称+新盘探盘」，可修改 | ✅ |
| 优先级 | 高/中/低，默认中 | ✅ |
| 截止时间 | 日期时间选择器 | ✅ |
| 探盘要求 | 填写具体要求 | ✅ |
| 定位打卡 | 是否需要现场定位 | ❌ |
| 必填点位 | 拍照/视频/录音/文本，至少选一项 | ✅ |
| 选填点位 | 同上，可多选或不选 | ❌ |

**说明：** 配置内容与PC端保持一致

## 六、与PC端下发关系

### 6.1 数据共用

**小程序下发的任务和PC下发的任务共用数据**，同一张任务表，可在任意一端查看和操作。

| 维度 | PC端 | 小程序端 |
|------|------|----------|
| 入口 | PC任务列表 → 新建 | 小程序通知 → 楼盘详情 |
| 表单 | 完整表单（可多楼盘） | 简化表单（单楼盘） |
| 楼盘选择 | 下拉选择已有楼盘 | 直接展示上线楼盘 |
| 执行人员 | 下拉多选 | 勾选列表（组织架构） |
| 适用场景 | 日常管理、批量操作 | 移动办公、应急下发 |
| 审核流程 | 无需审核，直接下发 | 无需审核，直接下发 |

### 6.2 任务来源标识

任务列表中通过标签区分来源：

| 来源 | 标签样式 |
|------|----------|
| PC下发 | 灰色标签「PC」 |
| 小程序下发 | 橙色标签「小程序」 |

### 6.3 审核说明

**探盘任务下发无需审核**，配置完成后直接下发到执行人员。

## 九、验收标准

| 功能 | 验收条件 |
|------|----------|
| 新房通知 | PC配置可下发人员后，小程序收到通知 |
| 楼盘详情 | 显示楼盘基本信息（同PC） |
| 配置表单 | 表单完整，与PC一致，必填项校验 |
| 执行人员 | 按组织架构筛选，支持搜索 |
| 确认下发 | 直接下发，无需审核，状态同步 |
| 通知联动 | 下发后执行人员收到任务通知 |
| 数据共用 | 与PC下发任务共用数据，列表统一展示 |
