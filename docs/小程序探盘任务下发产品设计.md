## 一、产品定位

### 1.1 功能定位

让**区域经理/业务线总监**在小程序侧也能完成探盘任务下发，无需打开PC端，实现移动化办公。

### 1.2 核心场景

| 场景 | 描述 |
|------|------|
| 收到通知 | 新房项目上线后，区域经理收到推送 |
| 查看楼盘 | 点击通知查看楼盘信息 |
| 下发任务 | 选择楼盘后，配置执行人员并下发 |

### 1.3 与PC端差异

| 维度 | PC端 | 小程序端 |
|------|------|----------|
| 适用角色 | 运营人员、管理员 | 区域经理、业务线总监 |
| 操作复杂度 | 完整表单 | 简化下发流程 |
| 场景 | 日常管理 | 移动办公、应急下发 |

---

## 二、用户角色

| 角色 | 操作 |
|------|------|
| 区域经理 | 收到通知 → 查看楼盘 → 下发任务 |
| 业务线总监 | 同上，可批量下发 |

## 六、与PC端下发差异

| 维度 | PC端 | 小程序端 |
|------|------|----------|
| 入口 | PC任务列表 → 新建 | 小程序通知 → 楼盘详情 |
| 表单 | 完整表单（可多楼盘） | 简化表单（单楼盘） |
| 楼盘选择 | 下拉选择已有楼盘 | 直接展示上线楼盘 |
| 执行人员 | 下拉多选 | 勾选列表 |
| 适用场景 | 日常管理、批量操作 | 移动办公、应急下发 |

---

## 七、交互汇总

| 入口 | 行为 |
|------|------|
| 通知铃铛 | 展开通知面板 |
| 新房上线通知 | 标记已读 + 跳转楼盘详情 |
| 查看楼盘信息 | 展示楼盘基本信息 |
| 配置探盘任务 | 填写任务配置表单 |
| 选择执行人员 | 勾选执行人员 |
| 确认下发 | 下发任务，状态同步 |

## 八、数据结构

### 8.1 新房上线通知

```javascript
{
  notifyId: "NOTIFY_xxx",
  type: "house_online",  // 新房上线
  houseId: "HOUSE_xxx",
  houseName: "中海汇德里",
  developer: "中海地产",
  message: "新房项目已上线，请配置探盘任务",
  isRead: false,
  createTime: "2026-03-31T21:00:00Z"
}
```

### 8.2 探盘任务下发请求

```javascript
{
  taskName: "中海汇德里新盘探盘",
  houseId: "HOUSE_xxx",
  priority: "medium",  // high/medium/low
  
  // 时间
  deadline: "2026-04-07T18:00:00Z",
  
  // 配置
  requireCheckIn: true,
  requiredPoints: ["photo", "video"],
  optionalPoints: [],
  
  // 探盘要求
  requirement: "请拍摄沙盘全景...",
  
  // 执行人员
  executorIds: ["USER_001", "USER_002", "USER_003"]
}
```

| 功能 | 验收条件 |
