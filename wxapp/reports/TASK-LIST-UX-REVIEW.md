## 三、合并方案分析

### 3.1 方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|---------|
| **A. 合并为一个页面** | 减少页面，统一入口 | 功能堆砌，复杂度增加 | ⭐⭐ |
| **B. 保留分离，增强差异** | 各司其职，专注体验 | 页面数量多 | ⭐⭐⭐⭐ |
| **C. 合并入口，保留视图** | 统一入口，多视图切换 | 实现成本中等 | ⭐⭐⭐⭐⭐ |

### 3.2 推荐方案：C - 合并入口，保留视图

**核心理念：**
- 保留两个页面作为不同"视图模式"
- 在 task-list.html 添加"视图切换"入口
- calendar.html 可视为 task-list.html 的"日历视图模式"

**实现方式：**

```
task-list.html (任务列表页)
    │
    ├── 视图切换: 列表视图 / 日历视图
    │
    ├── 列表视图 (当前设计)
    │       - Tab: 全部/探盘/话术/内容
    │       - Chip: 状态筛选
    │       - 任务卡片列表
    │
    └── 日历视图 (跳转至 calendar.html)
            - 左侧: 日期列表
            - 右侧: 当日任务
            - 筛选栏独立一行
```

## 六、修复建议

### 6.1 立即修复 (P0)

```html

<div class="task-card status-progress" onclick="location.href='task-detail.html'">

    <i class="fa fa-tasks"></i>

<!-- 3. 添加 Focus 样式 -->
<style>
button:focus-visible,
a:focus-visible {
    outline: 2px solid var(--brand-primary);
    outline-offset: 2px;
}
</style>
```

### 6.2 短期优化 (P1)


### 6.3 合并方案 (P2)

在 task-list.html 添加视图切换：

```html
<div class="view-toggle">
    <button class="view-btn active" onclick="switchView('list')">

    </button>
```
