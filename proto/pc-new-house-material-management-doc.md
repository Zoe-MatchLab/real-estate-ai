## 🧠 头脑风暴区
> 请在此处记录所有想法、需求、痛点、期望功能点：
### 用户痛点
- [x] 新盘创建和谈判任务脱节，需要重复录入信息
- [x] 新盘审核流程不规范，缺乏统一管控
- [x] 楼盘资料和物料分散管理，难以查找关联
### 核心需求
- [x] 统一展示所有楼盘完整资料
- [x] 新盘创建、审核全流程标准化
- [x] 对接现有谈判任务管理，支持从谈判页直接创建新盘
### 期望功能点
- [x] 新盘全生命周期管理（创建→审核→上线→维护）
- [x] 楼盘与物料关联管理
- [x] 审核流程可追溯
### 特殊场景
- [x] 谈判成功的项目直接转化为新盘，无需重复填写信息

## 📋 需求分析
### 功能范围
| 功能模块 | 优先级 | 描述 |
|----------|--------|------|
| 新盘管理列表 | ⭐⭐⭐⭐⭐ | 展示所有楼盘完整资料，支持多维度筛选搜索 |
| 新盘创建与编辑 | ⭐⭐⭐⭐⭐ | 新盘表单填写、草稿保存、上下架管理 |
| 谈判任务联动 | ⭐⭐⭐⭐⭐ | 对接现有谈判任务页，支持直接从谈判创建新盘，信息自动同步 |
| 楼盘物料关联管理 | ⭐⭐⭐⭐ | 每个楼盘下关联管理所属所有物料 |
| 批量操作 | ⭐⭐⭐ | 批量上下架、批量导出 |
| 操作日志 | ⭐⭐⭐ | 全流程操作可追溯 |
| 权限控制 | ⭐⭐⭐ | 不同角色差异化权限 |

### 用户角色
- 运营人员：可创建、编辑、上下架新盘，管理物料
- 超级管理员：全功能权限，可删除楼盘
- 普通人员：仅查看权限

## 🎨 功能设计
### 1. 新盘管理列表页（展示所有楼盘资料）
#### 页面结构
- [ ] 顶部筛选栏：区域筛选、价格区间筛选、楼盘状态筛选（已上线/已下架）、创建时间筛选、关键词搜索（楼盘名称、开发商、项目ID）
- [ ] 视图切换：卡片视图/表格视图切换按钮
- [ ] 列表展示区：
  - 卡片视图：楼盘封面、名称、区域、均价、状态标签、关联物料数、操作按钮
  - 表格视图：楼盘ID、名称、区域、均价、户型区间、开发商、上线状态、关联谈判任务ID、关联物料数、更新时间、操作按钮
- [ ] 分页控件
#### 操作项（右上角按钮并排显示）
- ✅ **新建楼盘**：蓝色主按钮，点击打开新盘创建表单
- ✅ **上传新盘资料**：白色边框按钮，点击上传资料
- 查看详情
- 编辑
- 上下架
- 关联物料
- 删除（仅管理员可见，二次确认）

---
### 2. 新盘创建功能
#### 创建表单字段
- [ ] 基础信息模块：楼盘名称、所在区域、详细地址、均价、户型区间、开发商、物业公司、容积率、绿化率、装修情况
- [ ] 区位配套模块：地图标注位置、交通配套、教育配套、医疗配套、商业配套
- [ ] 销售信息模块：开盘时间、交房时间、优惠政策、销售状态
- [ ] 资料上传模块：封面图、效果图、户型图、宣传视频、VR链接、其他附件
- [ ] 关联信息模块：关联谈判任务（可选）、标签设置
#### 交互特性
- [ ] 草稿自动保存，退出时提示是否恢复
- [ ] 表单实时校验，必填项标红提示
- [ ] 图片上传支持拖拽、预览、裁剪

---
### 3. 谈判任务联动功能
- [ ] 探盘任务管理页面新增「创建为新盘项目」按钮（谈判成功的任务显示）
- [ ] 点击后直接跳转新盘创建页，自动带入谈判中的楼盘信息（名称、开发商、价格、户型等），无需重复填写
- [ ] 新盘创建成功后与对应谈判任务双向绑定，可互相跳转查看

---
### 4. 楼盘物料关联管理
- [ ] 每个楼盘详情页下设「物料管理」标签页
- [ ] 支持上传、关联、移除该楼盘所属的所有物料（户型图、VR、宣传资料、销售物料等）
- [ ] 物料分类展示，支持搜索筛选

---
## 🎯 交互设计说明
### 核心交互
1. 上下架操作实时生效，无需二次刷新
2. 新盘编辑支持草稿保存，退出自动提示
3. 批量操作支持：批量上下架、批量删除
4. 图片上传支持拖拽上传、预览、裁剪

### 异常处理
- 楼盘删除前校验是否被其他地方引用
- 重复楼盘名称提示
- 资料上传格式校验提示
